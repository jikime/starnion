# jiki (지기) - 제품 개요

**최종 수정일:** 2026-03-01
**버전:** 0.1.0
**상태:** Planning

---

## 비전

> "나보다 나를 더 잘 아는 디지털 트윈(Digital Twin)"

jiki(지기)는 知己(지기, "나를 진정으로 아는 존재")에서 이름을 따온 초개인화 AI 에이전트 플랫폼이다. 사용자의 일상 데이터를 축적하고 학습하여, **의사결정 피로(Decision Fatigue)**를 근본적으로 해소하는 것이 핵심 가치이다.

---

## 핵심 가치 제안

### 의사결정 피로 해소

현대인은 하루 평균 35,000건 이상의 의사결정을 내린다. jiki는 사용자의 패턴, 선호도, 목표를 학습하여 반복적이고 사소한 결정들을 자동화하거나 최적의 선택지를 제안한다.

### 데이터 축적 효과 (Learning Effect)

- 사용할수록 추천 정확도가 향상되는 구조
- 구독 모델의 자연스러운 정당화: 데이터가 쌓일수록 이탈 비용 증가
- 개인 맥락 기반 의사결정 지원으로 범용 AI와 차별화

### Privacy-First 접근

- 모든 개인 데이터는 사용자 소유 원칙
- 구조화된 데이터 저장으로 투명한 데이터 관리
- 벡터 임베딩 기반 검색으로 원본 데이터 노출 최소화

## 핵심 기능

### 1. 자연어 데이터 입력

사용자가 텔레그램으로 일상적인 메시지를 보내면, LLM이 자동으로 데이터를 파싱하고 구조화하여 저장한다.

**흐름:**

```
사용자: "오늘 점심 만원 썼어"
    ↓ LLM 파싱
카테고리: 식비 | 금액: 10,000원 | 날짜: 오늘 | 설명: 점심
    ↓ Tool 호출
save_finance(category="식비", amount=10000, description="점심")
    ↓
응답: "점심 식비 10,000원 기록했어요. 이번 달 식비 누적 230,000원이에요."
```

**지원 시나리오:**

| 입력 예시 | 파싱 결과 |
|----------|----------|
| "오늘 점심 만원 썼어" | 식비 10,000원 |
| "스타벅스 아이스아메리카노 4,500원" | 카페 4,500원 |
| "월급 350 들어왔어" | 수입 3,500,000원 |
| "택시비 12,000원" | 교통 12,000원 |
| "넷플릭스 구독료 빠져나갔어" | 구독 (금액 조회 필요) |

### 2. 3계층 메모리 시스템

jiki의 개인화 능력의 핵심은 3계층으로 설계된 메모리 시스템이다.

| 계층 | 저장소 | 용도 | 수명 |
|------|--------|------|------|
| **단기 기억** | LangGraph State | 현재 대화 맥락 유지 | 세션 단위 |
| **장기 기억** | pgvector RAG | 과거 대화, 사용자 이력 검색 | 영구 |
| **엔티티 기억** | Structured JSON (profiles) | 선호도, 목표, 습관 등 프로필 | 영구 (갱신) |

**메모리 상호작용 예시:**

```
사용자: "오늘 저녁 뭐 먹을까?"
    ↓
단기 기억: 오늘 점심에 한식을 먹었음
장기 기억: 최근 2주간 일식을 자주 먹음, 매운 음식 선호
엔티티 기억: 유제품 알레르기, 저녁 예산 보통 15,000원
    ↓
응답: "점심에 한식 드셨으니, 오늘 저녁은 가벼운 일본 라멘 어때요?
       근처 '멘야하나비'가 평점 4.5에 매운 라멘이 인기메뉴예요.
       보통 13,000원 정도면 돼요."
```

### 3. Tool Use (자율적 액션)

LangGraph의 ReAct 패턴을 활용하여, AI가 상황에 맞는 도구를 자율적으로 선택하고 실행한다.

| Tool | 기능 | 호출 예시 |
|------|------|----------|
| `save_finance` | 수입/지출 기록 | 금액, 카테고리, 설명 저장 |
| `get_monthly_total` | 월별 집계 조회 | 카테고리별 합계, 예산 대비 |
| `query_knowledge_base` | 개인 지식 검색 | 문서, 메모, 과거 대화 RAG 검색 |
| `manage_calendar` | 일정 관리 | 일정 추가, 조회, 알림 설정 |

**ReAct 패턴 동작:**

```
사용자: "이번 달 식비 얼마 썼어?"

Plan: 월별 식비 합계를 조회해야 함
Execute: get_monthly_total(category="식비", month="2026-03")
Observe: 총 230,000원, 예산 300,000원 대비 76.7%

응답: "이번 달 식비는 230,000원이에요.
       예산 300,000원 대비 76.7% 사용했고, 잔여 70,000원 남았어요.
       현재 페이스라면 월말까지 약 310,000원으로 소폭 초과 예상돼요."
```

### 4. 멀티모달 처리

텍스트 외에도 다양한 형태의 입력을 처리한다.

| 입력 유형 | 처리 방식 | 활용 예시 |
|----------|----------|----------|
| **이미지** | Receipt OCR, 음식 사진 인식 | 영수증 사진으로 자동 가계부 입력 |
| **음성** | STT (Speech-to-Text) | 음성 메시지로 메모/기록 |
| **문서** | PDF, DOCX, XLSX, HWP 파싱 | 개인 문서 지식베이스 구축 |

### 5. 개인 지식베이스 (Personal RAG)

사용자가 업로드한 문서를 청킹(chunking)하고 벡터 임베딩하여, 언제든 자연어로 검색할 수 있는 개인 지식베이스를 구축한다.

**파이프라인:**

```
문서 업로드 (PDF, DOCX 등)
    ↓ 파싱 & 청킹
문단 단위 분할 (500-1000 토큰)
    ↓ 벡터 임베딩
Gemini Embedding (1536차원)
    ↓ pgvector 저장
HNSW 인덱스로 고속 유사도 검색
    ↓
자연어 질의 → 관련 문단 검색 → LLM 답변 생성
```

**활용 시나리오:**

- "지난번 회의록에서 마케팅 예산 관련 내용 찾아줘"
- "내가 저장한 레시피 중에 닭가슴살 요리 있어?"
- "보험 계약서에서 해지 환급금 조건이 뭐였지?"

### 6. 프로액티브 알림

사용자가 요청하지 않아도, 데이터 기반으로 유용한 인사이트를 선제적으로 제공한다.

| 알림 유형 | 트리거 조건 | 메시지 예시 |
|----------|-----------|-----------|
| **예산 경고** | 카테고리별 예산 70%/90% 도달 | "이번 달 식비 예산의 70% 사용했어요" |
| **건강 인사이트** | 수면/운동 데이터 분석 | "어제 수면 점수가 낮으니 가벼운 식단 추천해요" |
| **주간 리포트** | 매주 일요일 | 지출 요약, 목표 달성률, 다음 주 제안 |
| **월간 리포트** | 매월 1일 | 월별 트렌드, 절약 포인트, 개선 제안 |
| **패턴 감지** | 비정상 지출 패턴 | "평소보다 교통비가 2배 높아요. 확인해보실래요?" |

### 7. 콜드 스타트 전략

신규 사용자의 초기 데이터 부족 문제를 해결하기 위한 3단계 전략이다.

| 단계 | 방법 | 수집 데이터 |
|------|------|-----------|
| **1단계** | 인터랙티브 온보딩 | 기본 프로필, 목표, 예산 |
| **2단계** | 외부 연동 | Google Calendar, 은행 앱 등 |
| **3단계** | 대화 기반 학습 | 자연스러운 대화에서 선호도 추출 |

**온보딩 대화 예시:**

```
jiki: "반가워요! 저는 지기예요. 몇 가지만 알려주시면 더 잘 도와드릴 수 있어요."
jiki: "매달 식비 예산은 어느 정도로 생각하고 계세요?"
사용자: "30만원 정도?"
jiki: "좋아요! 식비 예산 30만원으로 설정했어요. 교통비는요?"
```

## 아키텍처 개요

```
┌──────────────────────────────────────────────┐
│                사용자 인터페이스                  │
│  ┌──────────────┐     ┌──────────────────┐   │
│  │ Telegram Bot  │     │  Web UI (Next.js) │   │
│  │  (Primary)    │     │  (Dashboard)      │   │
│  └──────┬───────┘     └────────┬─────────┘   │
└─────────┼──────────────────────┼─────────────┘
          │ HTTP                  │ WebSocket/SSE
          ▼                      ▼
┌──────────────────────────────────────────────┐
│              Go Gateway (Echo v4)              │
│  Authentication │ Rate Limiting │ File Storage │
└──────────────────────┬───────────────────────┘
                       │ gRPC (Bidirectional Streaming)
                       ▼
┌──────────────────────────────────────────────┐
│         Python LangGraph Agent                 │
│  ReAct Pattern │ Tool Use │ Memory System      │
│  ┌────────┐ ┌────────┐ ┌──────────────────┐ │
│  │ Tools  │ │ Memory │ │ Knowledge Base   │ │
│  └────────┘ └────────┘ └──────────────────┘ │
└──────────────────────┬───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│         PostgreSQL + pgvector                  │
│  Profiles │ Finances │ Daily Logs │ Documents  │
│  Vector Embeddings (1536 dim) │ HNSW Index     │
└──────────────────────────────────────────────┘
```

### 통신 프로토콜

| 구간 | 프로토콜 | 목적 |
|------|---------|------|
| Telegram → Gateway | HTTP (Telegram API) | 봇 메시지 중계 |
| Web UI → Gateway | WebSocket, SSE | 실시간 채팅 스트리밍 |
| Gateway → Agent | gRPC (양방향 스트리밍) | 모델 추론 + Tool 호출 |
| Agent → Gateway | gRPC (ToolRegistry) | Tool 디스커버리 |

## 사용자 시나리오

### 시나리오 1: 일상 가계부 관리

```
[아침]
사용자: "출근길 커피 4,500원"
jiki: "카페 4,500원 기록했어요. ☕"

[점심]
사용자: (영수증 사진 전송)
jiki: "영수증 인식 결과: 김치찌개 8,000원, 음료 1,500원
       식비 총 9,500원 기록했어요."

[저녁]
사용자: "오늘 얼마 썼지?"
jiki: "오늘 총 지출: 14,000원
       - 카페: 4,500원
       - 식비: 9,500원
       이번 달 누적: 245,000원 (예산 대비 81.7%)"
```

### 시나리오 2: 개인 지식 관리

```
사용자: (보험 계약서 PDF 업로드)
jiki: "보험 계약서를 분석했어요. 12개 섹션, 45페이지 처리 완료.
       주요 내용: 보장 범위, 납입 기간, 해지 환급금 조건 등
       궁금한 내용이 있으면 언제든 물어보세요!"

[2주 후]
사용자: "보험 해지하면 환급금 얼마야?"
jiki: "계약서 기준, 현재 시점(가입 후 2년) 해지 시
       납입 보험료의 약 45%인 540,000원이 환급 예정이에요.
       3년 차부터는 60%로 올라가니 참고하세요."
```

### 시나리오 3: 프로액티브 건강 인사이트

```
[월요일 아침, 자동 알림]
jiki: "좋은 아침이에요! 주간 리포트 준비했어요.

       💰 지난주 지출: 187,000원 (전주 대비 -12%)
       🍽️ 식비가 가장 큰 비중 (43%)

       📊 특이사항: 금요일 저녁 외식 빈도가 높아요.
       주 1회로 줄이면 월 약 80,000원 절약 가능해요.

       이번 주 목표 설정할까요?"
```

## 수익 모델

### 구독 기반 (SaaS)

| 티어 | 월 가격 | 기능 |
|------|--------|------|
| **Free** | 0원 | 기본 가계부, 월 100회 메시지 |
| **Basic** | 4,900원 | 무제한 메시지, 기본 분석 |
| **Pro** | 9,900원 | 멀티모달, 지식베이스, 프로액티브 알림 |
| **Premium** | 19,900원 | 가족 공유, API 접근, 맞춤 리포트 |

### 핵심 지표

- **Retention Rate**: 데이터 축적 효과로 3개월 이후 이탈률 급감 예상
- **ARPU**: Basic/Pro 티어 중심, 월 7,000원 목표
- **LTV/CAC**: 데이터 록인 효과로 LTV 24개월+ 목표

## 로드맵

### Phase 1: MVP (3개월)

- Telegram 봇 기본 인터페이스
- 자연어 가계부 입력/조회
- 기본 프로필 관리
- 단일 사용자 지원

### Phase 2: 개인화 강화 (3개월)

- 3계층 메모리 시스템 완성
- 멀티모달 입력 (이미지 OCR, 음성)
- 프로액티브 알림 시스템
- 개인 지식베이스 (RAG)

### Phase 3: 확장 (3개월)

- Web UI 대시보드 (Next.js)
- 외부 서비스 연동 (Google Calendar, 은행)
- 가족/그룹 공유 기능
- 구독 결제 시스템

### Phase 4: 고도화 (지속)

- AI 모델 파인튜닝 (사용자별)
- 예측 분석 (지출 패턴, 건강 추세)
- 제3자 플러그인 생태계
- 다국어 지원
