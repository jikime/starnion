# 부재중 브리핑 (briefing)

## 트리거 감지

아래 표현이 포함된 메시지는 브리핑 요청으로 처리합니다.

- "무슨 일 있었어", "뭐 있었어", "무슨 일 있었나"
- "브리핑", "캐치업", "catch up", "catch-up"
- "내가 없는 동안", "자리 비운 동안", "잠깐 자리 비웠는데"
- "오늘 요약", "오늘 정리", "업데이트해줘"

## 실행 절차

브리핑 요청이 감지되면 아래 3개 레이어를 **가능한 한 병렬로** 처리합니다.

### 레이어 1 — 이메일 (Google skill 활성 시에만)

- `google_mail_list(query="is:unread", max_results=20)` 호출
- 수신한 메일 중 **중요도가 높은 것**(계약, 결제, 긴급, 미팅, 마감)을 LLM이 판단해 1-3건 요약
- Google skill이 비활성이거나 연동되지 않은 경우 이 레이어는 **조용히 생략** (오류 메시지 표시 안 함)

### 레이어 2 — 예산 현황

- `get_budget_status()` 호출
- 예산 초과 카테고리가 있으면 언급, 없으면 "지출은 정상 범위" 한 줄로 마무리
- 예산이 설정되지 않은 경우 이 레이어를 생략

### 레이어 3 — 최근 관심 주제 후속 보도

1. `retrieve_memory(query="최근 검색 뉴스 기사 기술", date_from="<현재 - 48시간>")` 호출
2. 결과에서 **뉴스·기술·사건 관련 검색어**를 최대 2개 추출
   - "날씨", "환율", "맛집" 같은 단발성 주제는 제외
3. 추출된 주제가 있으면 `web_search` 또는 `naver_search`로 최신 보도 검색
4. 의미 있는 후속 내용이 있으면 1-2줄 요약, 없으면 이 레이어를 생략

## 출력 형식

- **전체 길이**: 3-6문장, 짧고 자연스럽게
- **우선순위**: 이메일 중요 건 → 예산 이슈 → 후속 보도 순
- **생략 규칙**: 내용이 없는 레이어는 자연스럽게 생략 (빈 항목 표시 안 함)
- **이모지**: 각 레이어 앞에 1개씩 사용 가능 (📬 이메일, 💰 예산, 📰 뉴스)

### 출력 예시

```
📬 읽지 않은 메일 4통 중 계약 관련 건이 하나 있어요. 확인이 필요해 보입니다.
💰 이번 달 식비가 예산보다 좀 넘었네요 (120%).
📰 아침에 보셨던 AI 관련 기사 후속이 나왔어요 — OpenAI가 새 모델을 발표했다고 합니다.
```

### 내용이 없는 경우

```
별다른 이슈는 없었어요. 이메일도 조용하고, 지출도 정상 범위입니다.
```

## 주의 사항

- 각 레이어는 독립적으로 실패 가능 — 한 레이어가 오류여도 나머지는 계속 처리
- 메일 본문을 그대로 출력하지 않음 — 반드시 요약만 제공
- 후속 보도 검색은 최신성이 중요 — `web_search` 쿼리에 "최신", "오늘", 현재 날짜 포함 권장
