# Production Deployment (nginx + systemd)
{: .no_toc }

This guide covers running Starnion as **systemd services** on Rocky Linux / RHEL
and exposing it via an **nginx reverse proxy** with a custom domain and SSL.

## Prerequisites

| Item | Details |
|------|---------|
| OS | Rocky Linux 8/9 (or RHEL 8/9, AlmaLinux 8/9) |
| Run-as user | `root` (recommended for personal servers) |
| Install path | `/root/.starnion/` (already set up via `starnion setup`) |
| nginx | `sudo dnf install nginx` |
| Node.js / pnpm | `pnpm build` completed inside `~/.starnion/ui/` |

> **Note**: For multi-user servers you can create a dedicated `starnion` account instead.
> Replace `/root` with `/home/starnion` throughout, and set `User=starnion` in the service files.

---

## Step 1: Install systemd service files

Starnion runs as three independent services plus a target that groups them.

| File | Role |
|------|------|
| `starnion-gateway.service` | Go API server (:8080) |
| `starnion-agent.service` | Python AI engine (:50051 gRPC) |
| `starnion-ui.service` | Next.js web UI (:3893) |
| `starnion.target` | Groups all three services (≡ `starnion dev`) |

### Copy the files

```bash
sudo cp ~/.starnion/scripts/starnion-gateway.service \
        ~/.starnion/scripts/starnion-agent.service \
        ~/.starnion/scripts/starnion-ui.service \
        ~/.starnion/scripts/starnion.target \
        /etc/systemd/system/

sudo systemctl daemon-reload
```

> **💡** The service files are also available in the `scripts/` folder of the GitHub repository.

---

## Step 2: Create the gateway environment file

`starnion-gateway.service` reads environment variables from `/etc/starnion/gateway.env`.
Copy the `.env` generated by `starnion setup`:

```bash
sudo mkdir -p /etc/starnion
sudo cp ~/.starnion/docker/.env /etc/starnion/gateway.env
sudo chmod 640 /etc/starnion/gateway.env
```

## Step 3: Verify Node.js / pnpm paths

`starnion-ui.service` runs `pnpm start`.
When Node.js is installed via **nvm**, systemd does not inherit the nvm `PATH`,
so the paths must be specified explicitly in the service file.

```bash
# Find the actual paths
which node
which pnpm
```

**If using nvm** (e.g. `/root/.nvm/versions/node/v24.x.x/bin/node`):

```bash
NODE_PATH=$(dirname $(which node))

# Update ExecStart to the correct pnpm path
sudo sed -i "s|ExecStart=/usr/local/bin/pnpm|ExecStart=$(which pnpm)|" \
    /etc/systemd/system/starnion-ui.service

# Inject the nvm node directory into PATH
sudo sed -i "/^Environment=HOME=/a Environment=PATH=${NODE_PATH}:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin" \
    /etc/systemd/system/starnion-ui.service

sudo systemctl daemon-reload
```

## Step 4: SELinux configuration (Rocky Linux)

Executables inside `/root` (or any home directory) carry the `admin_home_t` SELinux context,
which prevents systemd (`init_t`) from executing them.
Apply the `bin_t` context to each binary once.

### Gateway binary

```bash
sudo chcon -t bin_t /root/.starnion/bin/starnion-gateway
```

### Agent Python binary

The venv `python3` is a symlink chain. You must apply `bin_t` to the **actual** binary at the end of the chain.
When installed via **uv**:

```bash
# Find the real binary
readlink -f /root/.starnion/agent/.venv/bin/python3

# Apply bin_t (replace path with the output above)
sudo chcon -t bin_t /root/.local/share/uv/python/cpython-3.13-linux-x86_64-gnu/bin/python3.13
```

When using the system Python:

```bash
sudo chcon -R -t bin_t /root/.starnion/agent/.venv/bin/
```

### UI Node.js binary (nvm)

```bash
sudo chcon -t bin_t $(which node)
```

### Persist across reboots

`chcon` labels reset on `restorecon` or relabelling. For permanent labels:

```bash
sudo semanage fcontext -a -t bin_t "/root/.starnion/bin(/.*)?"
sudo semanage fcontext -a -t bin_t "/root/.local/share/uv/python(/.*)?"
sudo restorecon -Rv /root/.starnion/bin/
sudo restorecon -Rv /root/.local/share/uv/python/
```

Or allow all user-home executables via SELinux boolean:

```bash
sudo setsebool -P allow_user_exec_content 1
```

## Step 5: Build the UI

A production build must exist before starting the service.

```bash
cd ~/.starnion/ui && pnpm install && pnpm build
```

A successful build creates `~/.starnion/ui/.next/`.

## Step 6: Enable and start services

```bash
# Enable all services at boot
sudo systemctl enable starnion.target

# Start now (all three services start together)
sudo systemctl start starnion.target

# Verify
systemctl status starnion-gateway starnion-agent starnion-ui --no-pager
```

### Expected status output

```
● starnion-gateway.service - Starnion Gateway
     Active: active (running) since ...
   Main PID: 12345 (starnion-gatewa)

● starnion-agent.service - Starnion Agent (LangGraph gRPC server)
     Active: active (running) ...

● starnion-ui.service - Starnion UI (Next.js)
     Active: active (running) ...
```

## Step 7: Configure nginx reverse proxy

### Install nginx

```bash
sudo dnf install -y nginx
sudo systemctl enable --now nginx
sudo firewall-cmd --permanent --add-service=http --add-service=https
sudo firewall-cmd --reload
```

### Deploy the nginx config

```bash
sudo cp ~/.starnion/scripts/nginx-lets-ai-kr.conf \
        /etc/nginx/conf.d/your-domain.com.conf

sudo sed -i 's/lets\.ai\.kr/your-domain.com/g' \
        /etc/nginx/conf.d/your-domain.com.conf

sudo nginx -t
sudo systemctl reload nginx
```

### Key nginx config sections explained

```nginx
map $http_upgrade $connection_upgrade {
    default  upgrade;
    ''       "";
}

upstream starnion_ui {
    server 127.0.0.1:3893;
    keepalive 64;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSE (streaming chat) — must disable buffering and gzip
    location = /api/chat {
        proxy_pass         http://starnion_ui;
        proxy_buffering    off;
        proxy_read_timeout 300s;
        gzip               off;
        add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;
        add_header X-Frame-Options           "SAMEORIGIN"                          always;
        add_header X-Content-Type-Options    "nosniff"                             always;
        add_header Referrer-Policy           "strict-origin-when-cross-origin"     always;
    }

    # WebSocket
    location = /api/ws {
        proxy_pass         http://starnion_ui;
        proxy_buffering    off;
        proxy_read_timeout 3600s;
        proxy_set_header   Upgrade    $http_upgrade;
        proxy_set_header   Connection $connection_upgrade;
    }

    # Static assets (long-lived cache)
    location /_next/static/ {
        proxy_pass http://starnion_ui;
        add_header Cache-Control "public, max-age=31536000, immutable" always;
    }

    location / {
        proxy_pass http://starnion_ui;
    }
}
```

## Step 8: SSL certificate with Let's Encrypt

## Operations reference

### Service management

```bash
sudo systemctl start   starnion.target
sudo systemctl stop    starnion.target
sudo systemctl restart starnion.target

sudo systemctl restart starnion-gateway
sudo systemctl restart starnion-agent
sudo systemctl restart starnion-ui
```

### Log inspection

```bash
# Follow all services simultaneously
journalctl -u starnion-gateway -u starnion-agent -u starnion-ui -f --no-pager

# Last 100 lines from a single service
journalctl -u starnion-gateway -n 100 --no-pager

# Today's logs only
journalctl -u starnion-agent --since today --no-pager

# nginx logs
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

### nginx management

```bash
sudo nginx -t                # Syntax check
sudo systemctl reload nginx  # Zero-downtime config reload
sudo systemctl restart nginx # Full restart
```

## Troubleshooting

### Service fails to start

```bash
journalctl -u starnion-gateway -n 50 --no-pager
journalctl -u starnion-agent -n 50 --no-pager
journalctl -u starnion-ui -n 50 --no-pager
```

**Common causes:**

| Symptom | Cause | Fix |
|---------|-------|-----|
| `status=203/EXEC` | SELinux blocking (`admin_home_t`) | `chcon -t bin_t <binary path>` — see Step 4 |
| `Failed to load environment files` | `/etc/starnion/gateway.env` missing | Create the file — see Step 2 |
| `status=127` / `pnpm: not found` | nvm PATH not set in systemd | Add `Environment=PATH=...` to service file — see Step 3 |
| `Cannot find module .../server.js` | No standalone build | Use `pnpm start` as `ExecStart` |
| `exec format error` | Wrong binary architecture | Re-download the correct platform binary |
| `Address already in use` | Port conflict | `lsof -i :<port>` to find the process |

### SELinux denial check

```bash
sudo ausearch -m avc -ts recent --no-pager
```

### Finding the real uv Python binary

```bash
# Trace the symlink chain to the actual binary
readlink -f ~/.starnion/agent/.venv/bin/python3
```

### 502 Bad Gateway from nginx

```bash
systemctl status starnion-ui --no-pager
ls ~/.starnion/ui/.next/
ss -tlnp | grep 3893
```

### Chat responses stop mid-stream

```bash
sudo grep -A5 "location = /api/chat" /etc/nginx/conf.d/your-domain.com.conf
# Must contain:  proxy_buffering off;
# Must contain:  proxy_read_timeout 300s;
```
