## Native Development

Runs all services as local processes. Ideal for development and hacking on StarPion.

### Prerequisites

- Go 1.22+
- Python 3.13+ with [uv](https://docs.astral.sh/uv/)
- Node.js 22+ with [pnpm](https://pnpm.io/)
- PostgreSQL 16+ with pgvector
- MinIO

### Setup

```bash
starpion setup   # one-time configuration wizard
starpion dev     # start all services concurrently
```

`starpion dev` starts three processes with colored output:

```
[gateway]  Listening on :8080
[agent]    gRPC server listening on :50051
[ui]       ready on http://localhost:3000
```

Press `Ctrl+C` to stop all services.

### Individual services

```bash
starpion gateway          # API server only
starpion agent            # AI agent only
starpion ui               # Web UI only
starpion gateway --dev    # Gateway with hot-reload
starpion ui --dev         # UI with next dev
```

### Dependency auto-install

`starpion agent` automatically runs `uv sync` when `pyproject.toml` is newer than `.venv`.
`starpion ui` automatically runs `pnpm install` when `pnpm-lock.yaml` is newer than `node_modules`.

## Docker Compose

Runs the full stack in containers — recommended for self-hosted production deployments.

### Services

| Container | Image | Port |
|-----------|-------|------|
| `starpion-postgres` | pgvector/pgvector:pg16 | 5432 |
| `starpion-minio` | minio/minio:latest | 9000, 9001 |
| `starpion-agent` | built locally | 50051 |
| `starpion-gateway` | built locally | 8080 |
| `starpion-ui` | built locally | 3000 |

### Quick start

```bash
# Option A: via CLI (reads ~/.config/starpion/config.yaml)
starpion setup            # if not done yet
starpion docker setup     # generates docker/.env and starts services

# Option B: via bash script (standalone, no CLI required)
cd docker && bash setup.sh
```

### Day-to-day management

```bash
starpion docker ps               # list running containers
starpion docker logs -f          # stream all logs
starpion docker logs -f gateway  # stream one service
starpion docker restart gateway  # restart a service
starpion docker down             # stop all services
starpion docker down --volumes   # stop + delete data (DESTRUCTIVE)
```

Or with `make`:

```bash
make -C gateway docker-up
make -C gateway docker-logs
make -C gateway docker-down
```

### Environment variables

The `docker/.env` file is auto-generated by `starpion docker setup`. You can also copy and edit:

```bash
cp docker/.env.example docker/.env
# Edit docker/.env with your values
```

| Variable | Default | Description |
|----------|---------|-------------|
| `POSTGRES_PASSWORD` | *(required)* | PostgreSQL password |
| `POSTGRES_DB` | `starpion` | Database name |
| `POSTGRES_USER` | `starpion` | Database user |
| `MINIO_SECRET_KEY` | *(required)* | MinIO secret key |
| `MINIO_ACCESS_KEY` | `starpion` | MinIO access key |
| `MINIO_BUCKET` | `starpion-files` | MinIO bucket |
| `MINIO_PUBLIC_URL` | `http://localhost:9000` | External MinIO URL |
| `JWT_SECRET` | *(required, 32+ chars)* | JWT signing secret |
| `AUTH_SECRET` | *(required, 32+ chars)* | NextAuth secret |
| `GATEWAY_PORT` | `8080` | Gateway HTTP port |
| `GATEWAY_PUBLIC_URL` | `http://localhost:8080` | External gateway URL |
| `GRPC_PORT` | `50051` | Internal gRPC port |
| `UI_PORT` | `3000` | Web UI port |
| `NEXTAUTH_URL` | `http://localhost:3000` | NextAuth callback URL |

### Production checklist

- [ ] Set strong random values for all `*_SECRET` and `*_PASSWORD` variables
- [ ] Set `GATEWAY_PUBLIC_URL` and `NEXTAUTH_URL` to your actual domain
- [ ] Set `MINIO_PUBLIC_URL` to your MinIO domain or S3-compatible endpoint
- [ ] Configure a reverse proxy (nginx/Caddy) with TLS in front of the UI and Gateway
- [ ] Enable PostgreSQL backups
- [ ] Set up MinIO bucket replication or backup policy

### Reverse proxy example (Caddy)

```
starpion.example.com {
    reverse_proxy localhost:3000
}

api.starpion.example.com {
    reverse_proxy localhost:8080
}
```
