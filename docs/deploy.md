## Native Development

Runs all services as local processes. Ideal for development and hacking on StarNion.

### Prerequisites

- Go 1.22+
- Python 3.13+ with [uv](https://docs.astral.sh/uv/)
- Node.js 22+ with [pnpm](https://pnpm.io/)
- PostgreSQL 16+ with pgvector
- MinIO

### Setup

```bash
starnion setup   # one-time configuration wizard
starnion dev     # start all services concurrently
```

`starnion dev` starts three processes with colored output:

```
[gateway]  Listening on :8080
[agent]    gRPC server listening on :50051
[ui]       ready on http://localhost:3893
```

Press `Ctrl+C` to stop all services.

### Individual services

```bash
starnion gateway          # API server only
starnion agent            # AI agent only
starnion ui               # Web UI only
starnion gateway --dev    # Gateway with hot-reload
starnion ui --dev         # UI with next dev
```

### Dependency auto-install

`starnion agent` automatically runs `uv sync` when `pyproject.toml` is newer than `.venv`.
`starnion ui` automatically runs `pnpm install` when `pnpm-lock.yaml` is newer than `node_modules`.

## Docker Compose

Runs the full stack in containers — recommended for self-hosted production deployments.

### Services

| Container | Image | Port |
|-----------|-------|------|
| `starnion-postgres` | pgvector/pgvector:pg16 | 5432 |
| `starnion-minio` | minio/minio:latest | 9000, 9001 |
| `starnion-agent` | built locally | 50051 |
| `starnion-gateway` | built locally | 8080 |
| `starnion-ui` | built locally | 3000 |

### Quick start

```bash
# Option A: via CLI (reads ~/.config/starnion/config.yaml)
starnion setup            # if not done yet
starnion docker setup     # generates docker/.env and starts services

# Option B: via bash script (standalone, no CLI required)
cd docker && bash setup.sh
```

### Day-to-day management

```bash
starnion docker ps               # list running containers
starnion docker logs -f          # stream all logs
starnion docker logs -f gateway  # stream one service
starnion docker restart gateway  # restart a service
starnion docker down             # stop all services
starnion docker down --volumes   # stop + delete data (DESTRUCTIVE)
```

Or with `make`:

```bash
make -C gateway docker-up
make -C gateway docker-logs
make -C gateway docker-down
```

### Environment variables

The `docker/.env` file is auto-generated by `starnion docker setup`. You can also copy and edit:

```bash
cp docker/.env.example docker/.env
# Edit docker/.env with your values
```

| Variable | Default | Description |
|----------|---------|-------------|
| `POSTGRES_PASSWORD` | *(required)* | PostgreSQL password |
| `POSTGRES_DB` | `starnion` | Database name |
| `POSTGRES_USER` | `starnion` | Database user |
| `MINIO_SECRET_KEY` | *(required)* | MinIO secret key |
| `MINIO_ACCESS_KEY` | `starnion` | MinIO access key |
| `MINIO_BUCKET` | `starnion-files` | MinIO bucket |
| `MINIO_PUBLIC_URL` | `http://localhost:9000` | External MinIO URL |
| `JWT_SECRET` | *(required, 32+ chars)* | JWT signing secret |
| `AUTH_SECRET` | *(required, 32+ chars)* | NextAuth secret |
| `GATEWAY_PORT` | `8080` | Gateway HTTP port |
| `GATEWAY_PUBLIC_URL` | `http://localhost:8080` | External gateway URL |
| `GRPC_PORT` | `50051` | Internal gRPC port |
| `UI_PORT` | `3000` | Web UI port |
| `NEXTAUTH_URL` | `http://localhost:3893` | NextAuth callback URL |

### Production checklist

- [ ] Set strong random values for all `*_SECRET` and `*_PASSWORD` variables
- [ ] Set `GATEWAY_PUBLIC_URL` and `NEXTAUTH_URL` to your actual domain
- [ ] Set `MINIO_PUBLIC_URL` to your MinIO domain or S3-compatible endpoint
- [ ] Configure a reverse proxy (nginx/Caddy) with TLS in front of the UI and Gateway
- [ ] Enable PostgreSQL backups
- [ ] Set up MinIO bucket replication or backup policy

### Reverse proxy example (Caddy)

```
starnion.example.com {
    reverse_proxy localhost:3893
}

api.starnion.example.com {
    reverse_proxy localhost:8080
}
```

## Release Deployment

This section describes how to publish a new StarNion release to GitHub.

### Release architecture

When a release is built, goreleaser packages the following into each platform archive (`starnion_darwin_arm64.tar.gz`, etc.):

```
starnion              ← CLI binary (→ /usr/local/bin/)
starnion-gateway      ← Gateway HTTP server (→ ~/.starnion/bin/)
agent/                ← Python agent source (→ ~/.starnion/agent/)
  src/
  pyproject.toml
  uv.lock
ui/                   ← Next.js standalone build (→ ~/.starnion/ui/)
  server.js
  .next/
  public/
```

After installation via `install.sh`, the layout on a user's machine is:

```
/usr/local/bin/starnion        ← CLI
~/.starnion/
  bin/starnion-gateway         ← Gateway server binary
  agent/                       ← Python source (uv manages the venv)
  ui/                          ← Next.js standalone server
  starnion.yaml                ← Config (generated by starnion setup)
  ui/.env                      ← UI env vars (auto-refreshed on each start)
```

### Prerequisites (one-time)

**GitHub repository settings:**

- `Settings` → `Actions` → `General` → Workflow permissions → **Read and write permissions**
- `Settings` → `Pages` → Source: **Deploy from a branch** → branch `main`, folder `/docs`

### Publishing a release

```bash
# 1. Merge all changes to main
git checkout main && git pull

# 2. Create a semantic version tag
git tag v1.0.0

# 3. Push the tag — GitHub Actions takes over automatically
git push origin v1.0.0
```

GitHub Actions (`release.yml`) then:

1. Builds Go binaries for all platforms (darwin/linux × amd64/arm64)
2. Builds the Next.js UI in standalone mode (`NEXT_STANDALONE=1 pnpm build`)
3. Assembles `dist/staging/` with agent source and UI output
4. goreleaser produces four archives and uploads them to GitHub Releases
5. Automatically updates the version pin in `docs/install.sh` and pushes to main (→ GitHub Pages)

### Local release dry-run

Test the full build without publishing:

```bash
# Install goreleaser (one-time)
brew install goreleaser   # macOS
# or: go install github.com/goreleaser/goreleaser/v2@latest

# Snapshot build (no tag required, no upload)
goreleaser release --snapshot --clean

# Inspect the output
ls dist/
# starnion_darwin_arm64.tar.gz
# starnion_darwin_amd64.tar.gz
# starnion_linux_arm64.tar.gz
# starnion_linux_amd64.tar.gz
# checksums.txt
```
