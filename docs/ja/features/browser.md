# ブラウザ操作

## 概要

Starnion のブラウザ操作機能は、AI が実際の Chromium ブラウザを自動的に操作できるようにします。URL ナビゲーション、ボタンクリック、テキスト入力、ページスクリーンショットの撮影など、すべて自然言語の一言で実行できます。

スクリーンショットは自動的にチャットに画像として添付され、Telegram・Web チャットのどちらからでもすぐに確認できます。

---

## 有効化方法

ブラウザスキルはデフォルトで無効になっています。使用するには、サーバーに Playwright をインストールする必要があります。

```bash
# Playwright のインストール（agent コンテナ内またはネイティブ環境）
playwright install chromium
playwright install-deps chromium
```

インストール後、**設定 > スキル** で `browser` スキルを有効化してください。

## Headless / Headed モード

ブラウザが画面にウィンドウを表示するか（headed）、バックグラウンドで非表示に動作するか（headless）を設定します。

### 自動検出（デフォルト）

手動設定なしで、実行環境を分析して自動的にモードを決定します。

| 環境 | モード | 検出方法 |
|------|--------|----------|
| Docker コンテナ | Headless | `/.dockerenv` ファイルの存在を検出 |
| CI 環境（GitHub Actions 等） | Headless | `CI` 環境変数を検出 |
| Linux（ディスプレイなし） | Headless | `DISPLAY` / `WAYLAND_DISPLAY` 未設定 |
| macOS / Windows | Headed | デスクトップ環境 |
| Linux（ディスプレイあり） | Headed | `DISPLAY` 設定済み |

### 手動設定

自動検出を上書きするには、環境変数または設定ファイルを使用します。

**環境変数（最高優先度）：**

```bash
BROWSER_HEADLESS=false   # Headed を強制（ブラウザウィンドウを表示）
BROWSER_HEADLESS=true    # Headless を強制
```

**`~/.starnion/starnion.yaml`：**

```yaml
browser:
  headless: false   # Headed を強制
```

優先度：`BROWSER_HEADLESS` 環境変数 > `starnion.yaml` > 自動検出

## 使用例

### スクリーンショットの撮影

最も一般的な使い方です。URL を開いてスクリーンショットをチャットに画像として送信します。

```
ユーザー：Yahoo! Japan で今日の東京の天気を調べて
ボット：[ブラウザを操作中...]
       [スクリーンショット画像添付]
       東京の現在の天気のスクリーンショットです。

ユーザー：https://www.google.com/maps をキャプチャして
ボット：[ブラウザを操作中...]
       [地図スクリーンショット画像添付]
       Google マップのスクリーンショットです。
```

### Web ページのナビゲーションとクリック

```
ユーザー：Yahoo! Japan のログインページを開いて
ボット：Yahoo! Japan のログインページを開きました。タイトル：Yahoo! JAPAN

ユーザー：ログインボタンをクリックして
ボット：「ログイン」ボタンをクリックしました。
```

### テキスト入力

```
ユーザー：検索ボックスに「機械学習入門」と入力して
ボット：検索ボックスにテキストを入力しました。

ユーザー：Enter キーを押して
ボット：Enter キーを押しました。
```

## スクリーンショットの仕組み

高品質なスクリーンショットを実現するために、多段階のローディング検出を行います。

```
1. DOM 読み込み（domcontentloaded）
      ↓
2. ネットワークアイドル待機（最大 15 秒）
   — AJAX・API 呼び出しが完了するまで待機
      ↓
3. DOM 安定性検出（MutationObserver）
   — 800ms 間 DOM 変化がなければ「読み込み完了」と判断
   — 最大待機時間：12 秒
   — SPA の「ローディングスピナー → 結果表示」の切り替えを正確に検出
      ↓
4. 追加待機（オプション、wait_ms パラメータ）
   — 地図・チャートなど特殊なページに手動で追加待機を指定可能
      ↓
5. Canvas 検出 → 自動で 4 秒追加待機
   — 地図（Google Maps、Yahoo!マップ等）のタイル描画待機
      ↓
6. キャプチャして画像を送信
```

> **ヒント：** 地図のスクリーンショットでは、Canvas 自動検出によりタイルが十分に読み込まれてからキャプチャします。それでも不完全な場合は `wait_ms=3000` のように追加待機を指定できます。

## 利用可能なツール

AI が自動的に選択しますが、特定の操作を指示する際の参考にできます。

| ツール | 説明 | リクエスト例 |
|--------|------|-------------|
| `browser_open_screenshot` | URL を開く + スクリーンショット（ワンステップ） | 「この URL のスクリーンショットを撮って」 |
| `browser_navigate` | URL ナビゲーション | 「Google を開いて」 |
| `browser_screenshot` | 現在のページをスクリーンショット | 「今の画面を撮って」 |
| `browser_click` | 要素をクリック | 「OKボタンをクリックして」 |
| `browser_type` | テキスト入力 | 「検索ボックスに天気と入力して」 |
| `browser_press` | キー入力 | 「Enter を押して」 |
| `browser_hover` | 要素にホバー | 「メニューにマウスを乗せて」 |
| `browser_scroll` | スクロール | 「下にスクロールして」 |
| `browser_snapshot` | アクセシビリティツリーを取得 | 「ページの構造を教えて」 |
| `browser_wait_for` | 特定の要素が現れるまで待機 | （自動使用） |
| `browser_wait_ms` | 指定時間待機 | （自動使用） |
| `browser_get_text` | ページ全文テキスト抽出 | 「このページのテキストを取得して」 |
| `browser_evaluate` | JavaScript を実行 | （上級者向け） |
| `browser_close` | ブラウザを閉じる | 「ブラウザを閉じて」 |

## 検索 URL パターン

ホームページで検索ボックスをクリックするより、直接検索 URL を使う方が速く確実です。

| 検索エンジン | URL パターン |
|-------------|-------------|
| Yahoo! Japan | `https://search.yahoo.co.jp/search?p=キーワード` |
| Google | `https://www.google.com/search?q=キーワード` |
| Bing | `https://www.bing.com/search?q=キーワード` |

## セッション管理

- ブラウザセッションはユーザーごとに維持されます。
- 最後の使用から **5 分間** 操作がない場合、自動的に終了します。
- 作業完了後に「ブラウザを閉じて」と言うと、サーバーリソースを即座に解放できます。
- スクリーンショット撮影後もセッションは維持されるため、続けてクリックや入力などの操作が可能です。

## よくある質問

**Q. ログインが必要なサイトも操作できますか？**
できます。「ユーザー名の入力欄に user@example.com と入力して」「パスワード欄に入力して」のようにリクエストしてください。ただし、パスワードはチャット履歴に残るためご注意ください。

**Q. スクリーンショットがローディング中の画面になってしまう場合は？**
ほとんどの場合、DOM 安定性検出が自動的に解決します。特殊なページの場合は「3 秒後にスクリーンショットを撮って」のようにリクエストすると、AI が `wait_ms=3000` パラメータを使用します。

**Q. 地図のスクリーンショットが白い画面になってしまう場合は？**
Canvas 自動待機（4 秒）で対応できますが、ネットワークが遅い場合は不足することがあります。「5 秒待ってから地図のスクリーンショットを撮って」とリクエストしてみてください。

**Q. アンチボット対策でブロックされた場合は？**
一部のサイトは自動化アクセスをブロックします。この場合は `browser_get_text()` でテキストのみを抽出して内容を要約する方法で対応できます。

**Q. Docker 環境で headed モードを使用するには？**
Docker コンテナ内で headed モードを使用するには仮想ディスプレイ（Xvfb）が必要です。Docker 環境では headless モードの使用を推奨します。
