# メモ

## 概要

メモ機能は、覚えておきたい情報をすばやく保存し、タグで整理するシンプルなノートシステムです。日記が感情や日常を記録するパーソナルジャーナルだとすれば、メモは情報や事実を蓄積するパーソナルデータベースです。

**主な機能：**
- タグによる整理で素早く検索
- 最大100件のメモを保存
- AIが関連メモを自動で見つけてくれる
- タイトル＋内容のシンプルな構造でスッキリ管理

> メモはメモリスキルとは異なります。メモは明示的に保存・取得する構造化された情報ストアです。メモリはAIが会話のコンテキストを自動的に吸収する長期学習システムです。

## メモを書く

### Webで書く

1. サイドバーの **「メモ」** をクリックします。
2. 右上の **「+ 新規メモ」** ボタンをクリックします。
3. 以下を入力します：
   - **タイトル**（必須）：メモをすぐに識別できる名前
   - **内容**（任意）：メモの詳細な内容
   - **タグ**（任意）：分類カテゴリ（デフォルト：個人）
4. **「保存」** をクリックします。

保存したメモは最終更新日時の新着順でリストに表示されます。

### チャットで書く

チャット欄でAIにメモの保存を依頼すると即座に保存されます。

**会話例：**

> ユーザー：「牛乳と卵を買う必要があるのでメモしておいて」
>
> AI：「メモを保存しました！「買い物リスト」として記録しました。」

> ユーザー：「ミーティングメモを保存して — 来週火曜日にローンチ予定、マーケティングチームの承認が必要」
>
> AI：「ミーティングメモを保存しました。タグを「仕事」に設定しました。」

> ユーザー：「新しいアイデアのメモ — アプリにダークモードを追加できたらいいな」
>
> AI：「アイデアのメモを保存しました！「アイデア」タグで記録しました。」

**保存のトリガーフレーズ：**
- 「これをメモして」「これを記録して」「これを保存して」
- 「メモを残して —」「このメモを取っておいて」
- 「ミーティングメモを保存して」「Xを書き留めておいて」

## タグシステム

タグはメモを分類し、後から素早く見つけるための重要な機能です。

### デフォルトタグ

メモ保存時にタグを指定しなかった場合、**個人**タグが自動的に付与されます。AIがメモの内容を分析し、適切なタグを自動提案します：

| 内容の例 | 自動タグ |
|---------|---------|
| ミーティング内容、プロジェクト関連 | 仕事 |
| 買い物、食べ物のメモ | 個人 |
| 新しいアイデア、計画 | アイデア |
| 学習ノート、講義メモ | 学習 |
| 旅行計画、行きたい場所 | 旅行 |

### タグを追加する方法

**Webの場合：** タグフィールドに直接入力して保存
**チャットの場合：** 「仕事タグでこれをメモして」/ 「アイデアタグで保存して」

### タグで検索する

**Webの場合：** メモリストの上にあるタグフィルターボタンをクリック → 目的のタグを選択
**チャットの場合：**

> ユーザー：「仕事のメモだけ見せて」
>
> AI：「仕事タグのメモ：\n1. ミーティングメモ（3/5）\n2. プロジェクト計画（3/3）\n3. 締め切りスケジュール（2/28）」

## メモを検索する

### キーワード検索（Web）

メモ画面の検索バーで、タイトルや内容に含まれるキーワードで検索できます。大文字・小文字を区別せず、部分一致もサポートしています。

**例：** 「マーケティング」を検索 → タイトルや内容に「マーケティング」が含まれるすべてのメモが表示されます

### AIアシスト検索（チャット）

探しているものをAIに説明すると、関連するメモを見つけてくれます。正確なキーワードを覚えていなくても大丈夫です。

> ユーザー：「前回保存したミーティング関連のメモを見つけて」
>
> AI：「ミーティング関連のメモが見つかりました：\n1. 3/5 ミーティングメモ — 来週ローンチ予定\n2. 2/28 週次ミーティング — マーケティングチームのフィードバック」

> ユーザー：「プロジェクト関連のメモをすべて見せて」
>
> AI：「プロジェクト関連のメモが3件見つかりました：\n...」

> ユーザー：「食べ物に関するメモはありますか？」
>
> AI：「食べ物に関するメモがあります：\n- 買い物リスト：牛乳、卵\n- レストランリスト：パスタ屋さん（代々木）」

## AI活用例

メモ機能をAIと組み合わせたさまざまな使い方を紹介します。

### 素早くメモを保存する

```
ユーザー：「後で読むためにリンクを保存して — https://example.com/article」
AI：「リンクのメモを保存しました。タイトルを「参考リンク」に設定しました。」

ユーザー：「田中さんの電話番号 090-1234-5678 をメモとして保存して」
AI：「連絡先のメモを保存しました。「個人」タグで記録しました。」
```

### 構造化されたメモを保存する

```
ユーザー：「このミーティングメモを保存して：
  参加者：田中チームリーダー、鈴木副主任、佐藤担当
  議題：Q2マーケティング戦略
  決定事項：SNS広告予算を200,000円増額」
AI：「ミーティングメモを保存しました！「仕事」タグで記録しました。」
```

### メモを取り出す

```
ユーザー：「メモを見せて」
AI：「最近の5件のメモです：
  1. ミーティングメモ（仕事）— 3/8
  2. 買い物リスト（個人）— 3/7
  3. 旅行チェックリスト（旅行）— 3/5
  4. アイデアメモ（アイデア）— 3/3
  5. 学習まとめ（学習）— 3/1」

ユーザー：「旅行のメモを詳しく見せて」
AI：「[旅行チェックリストのメモ]
  パスポート、充電器、旅行保険証明書、現地SIMカード...」
```

### メモを削除する

```
ユーザー：「買い物リストのメモを削除して」
AI：「買い物リストのメモを削除しました。」
```

## 💬 テレグラム使用例

> ニオンに自然な言葉で話しかけるだけでOKです。特別なコマンドは不要 — 普段どおりにチャットしてください！

### メモを保存する

```
メモして：明日午後2時に歯医者の予約
```
→ 「予定」タグでメモが保存されます

```
旅行の持ち物メモ：パスポート、充電器、旅行保険
```
→ 「旅行」タグでメモが保存されます

### メモを確認する

```
メモ一覧を見せて
```
→ 保存されたメモの一覧を表示します

```
旅行関連のメモを探して
```
→ 「旅行」タグのメモを検索します

### メモを削除する

```
歯医者の予約メモを削除して
```
→ 該当するメモが削除されます

## ヒントとよくある質問

**Q. メモと日記をどう使い分ければいいですか？**

| 状況 | おすすめの機能 |
|------|--------------|
| 「今日どんな気分で何があったか」 | 日記 |
| 「後で参照する情報を保存したい」 | メモ |
| 「ミーティングメモ、ToDoリスト」 | メモ |
| 「感情的な一日を振り返りたい」 | 日記 |

**Q. メモは何件まで保存できますか？**
最大100件です。100件に達したら、新しいメモを保存する前に古いメモを削除する必要があります。

**Q. メモに複数のタグを付けられますか？**
現在、各メモには1つのタグしか付けられません。複数のトピックにまたがるメモの場合は、内容の中にキーワードを含めると検索時に見つけやすくなります。

**Q. チャットで保存したメモをWebでも見られますか？**
はい、チャットとWebは同じデータを共有しています。チャットで保存したメモはWebのメモリストにも表示され、確認・編集ができます。

**Q. メモを編集できますか？**
Webではメモをクリックすると編集画面に移動します。チャットでは現在直接編集はサポートされていないので、既存のメモを削除して新しいメモを保存してください。

**Q. メモの内容もAIのメモリに保存されますか？**
メモ自体はAIの長期記憶に直接保存されるわけではありません。ただし、会話中にメモを取り出したり内容に言及したりすると、その会話のコンテキストがAIのメモリに影響する場合があります。明示的に覚えてほしい場合は「これを覚えておいて」と伝えてください。
