# 日記

## 概要

Starnionの日記機能は、単純なテキストログを超えた、AIがあなたの感情や日常を理解・記憶するパーソナルジャーナルです。会話するように1日を話しかけると、AIが自動的に気分を分析して記録し、「あのときどんな気分だった？」と聞いたときに蘇らせてくれます。

> 日記はAIメモリのコアとなる情報源です。あなたが記録したことは、AIがあなたをより深く理解するために使われます。

---

## 日記を書く

### Webで書く

1. サイドバーの **「日記」** をクリックします。
2. 右上の **「+ 新規記録」** ボタンをクリックします。
3. タイトル（任意）、内容、感情タグを入力します。
4. **「保存」** をクリックします。

日付はデフォルトで今日になりますが、カレンダーアイコンをクリックして過去の日付に変更することもできます。

### チャットで書く

チャット欄に1日の出来事を自然に話しかけるだけで、AIが自動的に日記エントリとして記録します。特別なコマンドは不要です。

**会話例：**

> ユーザー：「今日はチームのプレゼンが思ったよりうまくいった。緊張したけど、ほっとしている。」
>
> AI：「うまくいってよかったです！緊張を乗り越えて素晴らしい結果を出しましたね。今日の日記に記録しました 😊」

> ユーザー：「今日の日記を書いて — 今朝は危うく寝坊しそうになって遅刻しかけたけど、ランチに新しいパスタ屋さんに行ったらとても美味しかった。」
>
> AI：「記録しました！パスタ屋さん、おいしそうですね — きっとまた行きたくなりますね。残りの一日も良い時間を。」

**ヒント：** 金額が含まれたメッセージ（例：「ランチに1,200円使った」）は自動的に支出エントリとして分類され、日記には保存されません。

## 日記エントリの構造

各日記エントリは以下のフィールドで構成されます：

| フィールド | 説明 | 例 |
|-----------|------|-----|
| **日付** | 書いた日付（YYYY-MM-DD） | 2025-03-08 |
| **タイトル** | 短いタイトル（任意） | チームプレゼン成功 |
| **内容** | エントリの本文 | 今日のプレゼンはとてもうまくいきました... |
| **感情** | AIが分類または手動で選択した気分 | すごく良い |
| **タグ** | トピック別の分類タグ | #仕事 #達成 |

### 感情の分類

AIは内容に基づいて各エントリを5つの感情のいずれかに自動分類します：

- **すごく良い** — 達成感、興奮、感激（「プレゼンが決まった！」「合格した！」）
- **良い** — 喜び、満足、感謝（「のんびりできた一日」「美味しい夕食」）
- **普通** — 特に感情の変化がない普通の日
- **疲れた** — 疲労、眠気、身体的な疲れ（「残業」「運動後にへとへと」）
- **悪い** — 悲しみ、怒り、失望、ストレス（「喧嘩した」「すごく残念」）

Webエディタで感情を手動で変更することもできます。

## 日記エントリを見る

### リスト表示

WebのDiary画面では、すべてのエントリが新着順に表示されます。1ページに50件が表示され、ページネーションで古いエントリに移動できます。

### 日付で検索する

月別フィルターを使って特定の月のエントリを表示します：

- 上部のドロップダウンから **年** と **月** を選択
- その月のすべてのエントリが新着順に表示されます

### チャットで閲覧する

> ユーザー：「先週の日記を見せて」
>
> AI：「先週のエントリです：\n- 3/3（月）：チームミーティング、気分：普通\n- 3/5（水）：友達と夕食、気分：良い\n- 3/7（金）：週末の締めくくり、気分：疲れた」

> ユーザー：「3月の日記をすべて見せて」
>
> AI：「3月の日記リストです。全部で12件あります...」

## AI分析

### 感情パターンを理解する

週次レポートにはその週の感情の流れが含まれます。日記データに基づいて、AIはどの感情が最も多く現れたか、特定の曜日にパターンがあるかを分析します。

**分析結果の例：**
```
今週の感情サマリー：
- すごく良い：2回、良い：3回、普通：1回、疲れた：1回
- 平日（月〜金）は疲れた感情が多く、週末は良い感情が多かった。
- 全体的な感情状態は先週と比べて改善しています！
```

### ストレスが高かった日を見つける

> ユーザー：「私がストレスを感じていたのはいつ？」
>
> AI：「「悪い」気分が記録されていた日：\n- 2025-02-14：残業、仕事のプレッシャー\n- 2025-02-28：重要なプレゼンに失敗\n- 2025-03-03：チームメンバーとの衝突」

## エントリをメモリとして活用する

日記に記録した内容はAIの長期記憶に保存されます。つまり、AIは関連する詳細を数日後、あるいは数ヶ月後でも覚えていて、状況に合った応答ができます。

**メモリ活用の例：**

> ユーザー：（3ヶ月後）「あのパスタ屋さんに行ったって言ったこと、覚えてる？」
>
> AI：「もちろんです！3月8日に新しくオープンしたパスタ屋さんに行って、とても美味しかったとおっしゃっていましたね。」

**メモリに保存されるもの：**
- 日記エントリで言及された場所、人、出来事
- 繰り返し現れる感情パターン
- ユーザーが重要と考えていること

**メモリに保存されないもの：**
- 金銭情報（家計簿に別途分類）
- 感情的な内容がなく事実の列挙だけのメモ（メモ機能を使用してください）

```
ユーザー: 今日初めて1時間フルに運動した。きつかったけど達成感がある。
AI: すごいですね！初めて1時間の運動を完走したんですね。その誇らしさが伝わってきます。
    今日のエントリを「すごく良い」の感情で記録しました。明日も頑張りましょう！ 💪

ユーザー: 先月、気分が良かった日はどんな日だった？
AI: 先月（2月）に「すごく良い」または「良い」で記録されたエントリ：
    - 2/8：久しぶりに友人と再会
    - 2/15：プロジェクトを完成させた
    - 2/22：家族の集まり

ユーザー: 2月の日記は何件あった？
AI: 2月の日記は18件あります。
    感情内訳：すごく良い 4、良い 7、普通 4、疲れた 2、悪い 1
```

## ヒントとよくある質問

**Q. 日記とメモの違いは何ですか？**
日記は感情や日常を記録するためのパーソナルジャーナルです。AIが感情を分析し、長期記憶に保存します。メモはタグベースの情報保管ツールで、感情分析は行いません。

**Q. 同じ日に複数の日記エントリを書けますか？**
はい、同じ日付に複数のエントリを書くことができます。1日の中でチャットを通じて複数回記録すると、それぞれが別の日記エントリとして保存されます。

**Q. 過去の日付で日記を書けますか？**
Webエディタで日付を直接変更して過去の日付でエントリを書くことができます。チャットでは「昨日こんなことがあった...」や日付を指定すると、その日付で記録されます。

**Q. 日記エントリを削除すると、AIのメモリからも消えますか？**
エントリを削除するとリストから削除されます。すでにAIメモリに保存された内容は、週次レポートが生成されたタイミングによって一部残る場合があります。

**Q. チャットから自動で日記として記録されるルールはありますか？**
感情や日常の体験を含むメッセージは自動的に日記エントリとして保存されます。金銭情報は支出データとして分類され、参考として残したい情報はメモとして分類されます。
