# レポート

## 概要

Starnionのレポート機能は、AIを使って日記、家計記録、目標、支出パターンを総合的に分析し、週次・月次のサマリーレポートを自動生成します。

毎週月曜日の午前9時（JST）に前週のアクティビティのサマリーがTelegramで自動送信されます。チャットで「週次レポートを作成して」と入力するといつでも即座にレポートを生成することもできます。生成されたすべてのレポート（過去のものも含む）はレポートメニューで確認できます。

> レポートは週次・月次の振り返りの出発点です。AIがデータを最初に整理するので、内容を確認して翌週の計画を立てましょう。

---

## レポートの種類

| 種類 | キー | 説明 | 自動生成時刻（JST） |
|------|-----|------|------------------|
| **週次レポート** | `weekly` | 過去7日間の総合分析 | 毎週月曜日 09:00 |
| **日次サマリー** | `daily` | 当日の支出とアクティビティのサマリー | 毎日 21:00（記録がある日のみ） |
| **月次レポート** | `monthly` | 当月の完全分析 | 毎月末日 21:00 |
| **支出異常** | `anomaly` | 平均の200%を超える支出のアラート | 3時間ごとに自動チェック |
| **パターンインサイト** | `pattern` | パーソナライズされた支出・行動パターンのインサイト | 毎日 14:00（条件が満たされた場合） |
| **目標進捗** | `goal` | アクティブな目標の進捗サマリー | 毎週水曜日 12:00 |

## 自動送信レポート

### 週次レポート — 毎週月曜日 09:00 JST

毎週月曜日の午前9時に、前週のアクティビティを分析したレポートがTelegram経由で自動送信されます。過去30日以内に家計記録があるユーザーにのみ送信されます。

**週次レポートの例：**
```
📊 2025年3月 第2週 — 週次レポート

💰 今週の総支出：18,740円

カテゴリ別支出：
🍽 飲食：       7,850円  (41.9%)
🚌 交通：       2,420円  (12.9%)
☕ カフェ：     1,870円  (10.0%)
🛒 ショッピング：4,200円  (22.4%)
🎬 娯楽：       2,400円  (12.8%)

先週比：+1,230円（▲7.0%）
月間累計：31,280円

💡 インサイト
カフェの支出が先週比35%増加しました。
飲食費は予算の87%（9,000円）を使用しています。
```

### 日次サマリー — 毎日 21:00 JST

当日支出を記録したユーザーに送信されます。

```
📋 日次サマリー — 2025年3月8日

本日の支出：4,230円
- 朝食：  650円
- ランチ：900円
- カフェ：450円
- 交通：  420円
- 夕食：1,810円

今週の累計：12,740円 / 30,000円（42.5%）
```

### 月次レポート — 毎月末日 21:00 JST

毎月末日の夜に送信され、当月の支出の完全分析を含みます。当月の記録があるユーザーに送信されます。

### 支出異常 — 3時間ごとの自動チェック

当日の支出が30日間の日次平均の200%を超えた場合、即座にアラートを送信します。有効化には少なくとも7日分のデータが必要です。

**支出異常の例：**
```
⚠️ 支出異常を検出しました

本日の総支出：18,500円
30日間の日次平均：3,120円  (593%)

通常より大幅に多い支出です。ご確認ください！
```

### パターンインサイト — 毎日 14:00 JST

AIが支出パターンを学習し、条件が満たされた場合にパーソナライズされたインサイトを送信します。パターン分析は毎朝6時にバックグラウンドで実行されます。

```
💡 パターンインサイト

金曜日の夜の支出が通常の2.3倍になっています。
過去4週間の金曜日の平均：4,820円
（平日の平均：2,100円）

金曜日の夜専用の予算を設定してみてはいかがでしょうか？
```

### 目標進捗 — 毎週水曜日 12:00 JST

アクティブな目標を持つユーザーに週次の進捗更新を送信します。

```
🎯 目標進捗（3月第3週）

✅ 読書チャレンジ：4/5冊（80%）— あと1冊！
🔄 運動習慣：8/12回（67%）— 今週あと2回必要
💰 旅行貯金：15,000円 / 20,000円（75%）

先週比の達成率：74%（+8%）
今週も頑張りましょう！ 💪
```

## レポートを手動で生成する

自動スケジュールを待たずに即座にレポートを生成できます。

### チャットでリクエストする

```
ユーザー：週次レポートを作成して。
AI:   今週のレポートを生成しました。[レポート内容が表示される]

ユーザー：今月の月次レポートを作成して。
AI:   3月の月次レポートを生成しました。

ユーザー：今日の支出パターンを分析して。
AI:   パターンインサイトレポートを生成しました。

ユーザー：貯蓄目標の進捗を教えて。
AI:   目標進捗レポートを生成しました。
```

### レポートメニューから生成する

上部メニュー > **「レポート」** タブ > **「レポートを生成」** ボタンをクリックし、希望のレポートタイプを選択します。生成には約20〜60秒かかります。

> 手動で生成したレポートもレポートリストに保存され、後から確認できます。

## レポートの詳細内容

### 週次レポート（weekly）

- 今週の総支出
- カテゴリ別の支出額と割合
- 先週比の増減
- 月間累計
- 予算消化率（予算が設定されている場合）
- AIインサイトと節約のヒント

### 月次レポート（monthly）

- 今月の総支出
- カテゴリ別の月間内訳
- 前月との比較
- 日別の支出トレンド
- 最も支出が多かった日と少なかった日
- 貯蓄目標を達成したかどうか

### パターンインサイト（pattern）

```
💡 支出パターンインサイト（2025-03-08）

1. カフェの支出パターン
   平日の朝（9〜11時）にカフェを利用する頻度が高い。
   週平均カフェ支出：2,350円

2. 週末の支出特性
   週末のショッピング支出は平日の3.2倍。
   週末前に買い物リストを作成することをお勧めします。

3. 節約の機会
   コンビニでの購入をスーパーでの買い物に変えると
   月に約1,500円の節約になります。
```

### 目標進捗（goal）

```
🎯 目標進捗（2025-03-08）

[目標1] 緊急資金として100,000円貯める
現在：65,000円 / 100,000円（65%）
完了予定日：2025年5月15日

[目標2] 月の飲食費を15,000円以内に収める
3月の飲食費：8,920円 / 15,000円（59.5%）
今月の目標達成の可能性：✅ 高い
```

## レポートを閲覧する

### レポートリスト

上部メニュー > **「レポート」** タブで、これまでに生成されたすべてのレポートを新着順で確認できます。一度に20件が表示され、「もっと見る」ボタンで追加のレポートを確認できます。

### レポートフィルター

| フィルター | 説明 |
|----------|------|
| すべて | すべてのレポート |
| 週次 | `weekly`タイプのみ |
| 月次 | `monthly`タイプのみ |
| パターン | `pattern`タイプのみ |
| 目標 | `goal`タイプのみ |
| 異常 | `anomaly`タイプのみ |

### 過去のレポートを確認する

```
ユーザー：先月の月次レポートを見せて。
AI:   2025年2月の月次レポートです：
      [2月のレポート内容が表示される]

ユーザー：先週の週次レポートと今週を比較して。
AI:   2月第4週 vs 3月第1週：
      総支出：17,520円 → 18,740円（+1,220円）
      最大増加：ショッピング（+1,800円）
      最大減少：飲食（−580円）
```

## 動作の仕組み

```
レポート生成フロー：

1. トリガー      → スケジューラー（自動）またはユーザーリクエスト（手動）
2. データ取得    → 日記、家計、目標、メモのデータを取得
3. AI分析       → gRPC経由でエージェントに分析リクエスト（最大120秒）
4. コンテンツ生成 → LLMが総合的なレポートテキストを生成
5. 保存          → レポートテーブルに永続的に保存
6. 配信          → Web UIに表示 + Telegram送信（Telegram接続時）
```

種類別のレポートタイトル形式：

| 種類 | タイトル例 |
|------|---------|
| `daily` | `日次サマリー — 2025年3月8日` |
| `weekly` | `2025年3月第2週 — 週次レポート` |
| `monthly` | `2025年3月 月次レポート` |
| `anomaly` | `2025-03-08 支出異常を検出` |
| `pattern` | `2025-03-08 パターンインサイト` |
| `goal` | `2025-03-08 目標進捗` |

## APIエンドポイント

### レポートの一覧取得

```
GET /api/v1/reports?user_id={user_id}&type={type}&limit=20&offset=0
```

| パラメータ | 必須 | 説明 |
|----------|------|------|
| `user_id` | 必須 | ユーザーID |
| `type` | 任意 | レポートタイプ（daily、weekly、monthly、anomaly、pattern、goal） |
| `limit` | 任意 | 返す最大件数（デフォルト：20） |
| `offset` | 任意 | オフセット（デフォルト：0） |

**レスポンス例：**
```json
[
  {
    "id": 42,
    "report_type": "weekly",
    "title": "2025年3月第2週 — 週次レポート",
    "created_at": "2025-03-10 09:00"
  }
]
```

### レポートの詳細取得

```
GET /api/v1/reports/{id}?user_id={user_id}
```

**レスポンス例：**
```json
{
  "id": 42,
  "report_type": "weekly",
  "title": "2025年3月第2週 — 週次レポート",
  "content": "📊 今週の支出サマリー...",
  "created_at": "2025-03-10 09:00"
}
```

### レポートの手動生成

```
POST /api/v1/reports/generate
Content-Type: application/json

{
  "user_id": "...",
  "report_type": "weekly"
}
```

`report_type`の有効値：`daily`、`weekly`、`monthly`、`anomaly`、`pattern`、`goal`

## 自動配信スケジュール一覧

| レポート | 配信時刻（JST） | 対象条件 |
|--------|--------------|---------|
| 週次レポート | 毎週月曜日 09:00 | 過去30日以内に家計記録があるユーザー |
| 日次サマリー | 毎日 21:00 | 当日に支出記録があるユーザー |
| 月次レポート | 毎月末日 21:00 | 当月に記録があるユーザー |
| 支出異常 | 3時間ごとに自動チェック | 支出が日次平均の200%を超えた場合 |
| パターンインサイト | 毎日 14:00 | パターントリガー条件が満たされた場合 |
| 目標進捗 | 毎週水曜日 12:00 | アクティブな目標があるユーザー |

> 通知過多を防ぐため、1日あたり最大3件の通知が送信されます。
> おやすみ時間（22:00〜08:00 JST）は通知が送信されません。

## よくある質問

**Q. 週次レポートを月曜日以外の日に受け取りたいです。**
A. 自動週次レポートの配信は現在毎週月曜日午前9時に固定されています。別の時間に受け取りたい場合は、チャットで直接「週次レポートを作成して」とリクエストしてください。

**Q. Telegramなしでもレポートを受け取れますか？**
A. 自動配信はTelegramのみ対応しています。Webでは、レポートメニューから直接レポートを生成するか、チャットでリクエストすることで確認できます。

**Q. 支出記録がなくてもレポートは生成されますか？**
A. 過去30日以内に支出記録がないユーザーは自動週次レポートの配信対象外です。手動でリクエストした場合は、「記録が見つかりませんでした」という内容のレポートが生成される場合があります。

**Q. 生成されたレポートはどのくらいの期間保持されますか？**
A. 生成されたすべてのレポートは永続的に保持されます。レポートメニューからいつでも過去のレポートを確認できます。

**Q. レポートの生成が遅すぎます。**
A. AIがすべての記録データを分析するため、通常20〜60秒かかります。記録が多い場合は処理時間が増加することがあります。

**Q. 支出異常のアラートが頻繁に来すぎます。**
A. 支出異常アラートは1日の通知上限（3件）を超えては送信されません。普段の支出が高めでしきい値が低く感じられる場合は、家計記録を積み上げ続けることで30日平均が実際の支出パターンをよりよく反映するようになり、誤ったアラートが減ります。
