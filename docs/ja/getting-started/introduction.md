## 概要

**Starnion** は完全にセルフホスティング可能なパーソナルAIアシスタントプラットフォームです。すべてのデータとAIのやり取りはご自身のインフラ上で動作し、外部サーバーにデータが送信されることは一切ありません。

クラウドAIサービスの利便性を保ちながら、個人情報とデータ主権を完全にコントロールできるよう設計されています。

---

## コアコンセプト

### パーソナルAIエージェント

StarnionのAIエージェントは、単純なチャットボット以上の存在です。LangGraphベースのグラフワークフローを通じて、複雑なタスクをステップごとに処理するインテリジェントなエージェントです。

- マルチAIプロバイダー対応（Anthropic Claude、OpenAI GPT、Google Gemini、Z.AI）
- スキルシステムによる機能拡張
- 長期記憶とコンテキスト管理
- ツールコールによる外部サービス連携

### プライバシーファースト

```
あなたのデータ = あなたのインフラのみに保存
```

- すべての会話履歴はご自身のPostgreSQLデータベースに保存
- AI APIコール時には必要最低限の情報のみを送信
- ファイル、画像、ドキュメントはご自身のMinIOストレージに保管
- サードパーティの分析サービスやトラッキングコードは一切なし

### セルフホスティング

```
自分のサーバーで動作 = 完全なコントロール
```

- スタック全体を単一のDocker Composeコマンドで起動可能
- クラウド、オンプレミス、ホームサーバーなど、どこでも動作
- インターネット接続なしでローカルAIモデルを使用可能（将来対応）
- データのバックアップとマイグレーションを直接コントロール

## 主要機能

| 機能 | 説明 |
|------|------|
| **マルチAIプロバイダー** | Anthropic、OpenAI、Google Gemini、Z.AIを同時サポート |
| **マルチチャンネル** | Web UI、Telegram、Discordからアクセス可能 |

| **音声処理** | ボイスメモのアップロードと文字起こし |
| **Web検索** | リアルタイムWeb検索の統合 |
| **ペルソナ** | ユーザー定義のAIパーソナリティとプロンプト |
| **Dockerサポート** | ワンコマンドデプロイ |
| **リアルタイムストリーミング** | WebSocketベースのリアルタイムレスポンスストリーミング |

```
┌─────────────────────────────────────────────────────────────┐
│                        Starnion                              │
│                                                             │
│  ┌──────────────┐    ┌───────────────┐    ┌─────────────┐  │
│  │      UI      │    │    Gateway    │    │    Agent    │  │
│  │  (Next.js)   │───▶│  (Go + REST)  │───▶│  (Python)   │  │
│  │   :3000      │    │    :8080      │    │   :50051    │  │
│  └──────────────┘    └──────┬────────┘    └──────┬──────┘  │
│                             │                    │         │
│                    ┌────────▼────────────────────▼──────┐  │
│                    │                                    │  │
│                    │  ┌──────────────┐ ┌────────────┐  │  │
│                    │  │  PostgreSQL  │ │   MinIO    │  │  │
│                    │  │ (+ pgvector) │ │ (Storage)  │  │  │
│                    │  │    :5432     │ │   :9000    │  │  │
│                    │  └──────────────┘ └────────────┘  │  │
│                    │         インフラストラクチャ層        │  │
│                    └────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘

チャンネル連携:
  Telegramボット ──▶ Gateway
  Discordボット  ──▶ Gateway
  Webブラウザ    ──▶ UI ──▶ Gateway
```

### コンポーネント説明

| コンポーネント | 役割 | 技術スタック |
|--------------|------|------------|
| **UI** | Webインターフェース | Next.js 15、React 19、TypeScript |
| **Gateway** | REST APIサーバー / WebSocket | Go 1.22+、Gin |
| **Agent** | AIエンジン / gRPCサーバー | Python 3.13+、LangGraph、gRPC |
| **PostgreSQL** | メインデータベース / ベクター検索 | PostgreSQL 16 + pgvector |
| **MinIO** | ファイルストレージ | MinIO（S3互換） |

### データフロー

```
ユーザーメッセージ
     │
     ▼
  UI (Next.js)
     │ HTTP / WebSocket
     ▼
  Gateway (Go)
     │ gRPC
     ▼
  Agent (Python / LangGraph)
     │
     ├──▶ AI API (Gemini / Claude / GPT)
     ├──▶ スキル実行（天気、検索、翻訳...）
     ├──▶ PostgreSQL（会話保存 / ベクター検索）
     └──▶ MinIO（ファイルアクセス）
     │
     ▼
  ストリーミングレスポンス → Gateway → UI
```

## 対象ユーザー

Starnionは以下のユーザーに適しています。

### 個人ユーザー

- AI会話をクラウドサービスに委ねずに完全にコントロールしたい方
- 個人の日記、メモ、目標をAIと一緒に管理したい方
- TelegramからどこでもパーソナルAIアシスタントにアクセスしたい方

### 開発者・技術ユーザー

- 独自のAIプラットフォームを構築したい開発者
- スキルシステムを通じてカスタム機能を追加したい方
- AIエージェントアーキテクチャを学びたい方

### 小規模チーム

- 社内AIアシスタントを運営したいチーム
- 顧客データを外部サービスに送りたくない企業
- 複数のチャンネル（Web、Telegram、Discord）でAIアクセスが必要なチーム

### 向いていない方

- サーバー管理なしですぐに使えるクラウドサービスが欲しい方 → ChatGPT、Claude.aiなどをお勧めします
- 数百人以上の同時ユーザーに対応する必要がある場合 → 追加のスケーリング作業が必要です
