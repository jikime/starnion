# 브라우저 제어

## 개요

Starnion의 브라우저 제어 기능은 AI가 실제 Chromium 브라우저를 자동으로 조작할 수 있게 합니다. URL 탐색, 버튼 클릭, 텍스트 입력, 페이지 스크린샷 캡처 등을 자연어 명령 하나로 수행합니다.

스크린샷은 자동으로 채팅에 이미지로 첨부되어 텔레그램·웹 채팅 모두에서 바로 확인할 수 있습니다.

---

## 활성화 방법

브라우저 스킬은 기본적으로 비활성화되어 있습니다. 사용하려면 서버에 Playwright가 설치되어 있어야 합니다.

```bash
# Playwright 설치 (에이전트 컨테이너 내부 또는 네이티브 환경)
playwright install chromium
playwright install-deps chromium
```

설치 후 **설정 > 스킬** 에서 `browser` 스킬을 활성화합니다.

## Headless / Headed 모드

브라우저가 화면에 창을 표시할지(headed) 여부를 설정합니다.

### 자동 감지 (기본)

별도 설정 없이도 실행 환경을 분석해 자동으로 결정합니다.

| 환경 | 모드 | 설명 |
|------|------|------|
| Docker 컨테이너 | Headless | `/.dockerenv` 존재 감지 |
| CI 환경 (GitHub Actions 등) | Headless | `CI` 환경변수 감지 |
| Linux (디스플레이 없음) | Headless | `DISPLAY` / `WAYLAND_DISPLAY` 미설정 |
| macOS / Windows | Headed | 데스크탑 환경 |
| Linux (디스플레이 있음) | Headed | `DISPLAY` 설정됨 |

### 수동 설정

자동 감지를 재정의하려면 환경변수 또는 설정 파일을 사용합니다.

**환경변수 (우선순위 최고):**

```bash
BROWSER_HEADLESS=false   # headed 강제 (브라우저 창 표시)
BROWSER_HEADLESS=true    # headless 강제
```

**`~/.starnion/starnion.yaml`:**

```yaml
browser:
  headless: false   # headed 강제
```

우선순위: `BROWSER_HEADLESS` 환경변수 > `starnion.yaml` > 자동 감지

## 사용 예시

### 스크린샷 찍기

가장 일반적인 사용 방법입니다. URL을 열고 스크린샷을 찍어 채팅에 이미지로 전송합니다.

```
나: 네이버에서 오늘 서울 날씨 검색해줘
봇: [브라우저 제어 중...]
    [스크린샷 이미지 첨부]
    서울 현재 날씨 스크린샷입니다.

나: https://www.google.com/maps 캡처해줘
봇: [브라우저 제어 중...]
    [지도 스크린샷 이미지 첨부]
    구글 지도 스크린샷입니다.
```

### 웹 페이지 탐색 및 클릭

```
나: 네이버 로그인 페이지 열어줘
봇: 네이버 로그인 페이지를 열었습니다. 제목: NAVER

나: 로그인 버튼 클릭해줘
봇: '로그인' 버튼을 클릭했습니다.
```

### 텍스트 입력

```
나: 검색창에 "기계학습 입문" 입력해줘
봇: 검색창에 텍스트를 입력했습니다.

나: Enter 눌러줘
봇: Enter 키를 입력했습니다.
```

## 스크린샷 동작 원리

스크린샷 품질을 높이기 위해 단계적 로딩 감지를 수행합니다.

```
1. DOM 로드 (domcontentloaded)
      ↓
2. networkidle 대기 (최대 15초)
   — AJAX·API 호출이 완료될 때까지 대기
      ↓
3. DOM 안정성 감지 (MutationObserver)
   — DOM 변화가 800ms 동안 없으면 "로딩 완료" 판단
   — 최대 대기: 12초
   — SPA 페이지의 "로딩 스피너 → 결과 표시" 전환을 정확히 감지
      ↓
4. 추가 대기 (선택, wait_ms 파라미터)
   — 지도·차트 등 특수 페이지에 수동 추가 대기 가능
      ↓
5. Canvas 감지 → 자동 추가 대기 4초
   — 지도(Google Maps, Naver Maps 등)의 타일 렌더링 대기
      ↓
6. 캡처 및 이미지 전송
```

> **팁:** 지도 스크린샷의 경우 Canvas 자동 감지로 타일이 충분히 로딩된 후 캡처합니다. 여전히 불완전하면 `wait_ms=3000` 같이 추가 대기를 지정할 수 있습니다.

## 사용 가능한 툴

AI가 자동으로 선택하지만, 특정 작업을 지시할 때 참고할 수 있습니다.

| 툴 | 설명 | 예시 요청 |
|----|------|----------|
| `browser_open_screenshot` | URL 열기 + 스크린샷 (원스텝) | "이 URL 스크린샷 찍어줘" |
| `browser_navigate` | URL 탐색 | "구글 열어줘" |
| `browser_screenshot` | 현재 페이지 스크린샷 | "지금 화면 찍어줘" |
| `browser_click` | 요소 클릭 | "확인 버튼 클릭해줘" |
| `browser_type` | 텍스트 입력 | "검색창에 날씨 입력해줘" |
| `browser_press` | 키 입력 | "Enter 눌러줘" |
| `browser_hover` | 요소 호버 | "메뉴 위에 마우스 올려줘" |
| `browser_scroll` | 스크롤 | "아래로 스크롤해줘" |
| `browser_snapshot` | 접근성 트리 조회 | "페이지 구조 알려줘" |
| `browser_wait_for` | 특정 요소 나타날 때까지 대기 | (자동 사용) |
| `browser_wait_ms` | 지정 시간 대기 | (자동 사용) |
| `browser_get_text` | 페이지 전체 텍스트 추출 | "이 페이지 텍스트 가져줘" |
| `browser_evaluate` | JavaScript 실행 | (고급 사용) |
| `browser_close` | 브라우저 종료 | "브라우저 닫아줘" |

## 검색 URL 패턴

검색 사이트는 홈페이지에서 검색창을 클릭하는 것보다 직접 검색 URL을 사용하는 게 빠르고 안정적입니다.

| 검색 엔진 | URL 패턴 |
|----------|----------|
| 네이버 | `https://search.naver.com/search.naver?query=검색어` |
| 구글 | `https://www.google.com/search?q=검색어` |
| 다음 | `https://search.daum.net/search?q=검색어` |

## 세션 관리

- 브라우저 세션은 사용자별로 유지됩니다.
- 마지막 사용 후 **5분** 동안 활동이 없으면 자동으로 종료됩니다.
- 작업 완료 후 "브라우저 닫아줘"로 즉시 종료하면 서버 리소스를 절약할 수 있습니다.
- 스크린샷 촬영 후에도 세션이 유지되므로 이어서 클릭·입력 등 인터랙션을 계속할 수 있습니다.

## FAQ

**Q. 로그인이 필요한 사이트도 제어할 수 있나요?**
가능합니다. "아이디 입력란에 user@example.com 입력해줘", "비밀번호 입력란에 입력해줘" 처럼 요청하면 됩니다. 단, 보안상 비밀번호는 채팅 히스토리에 남으니 주의하세요.

**Q. 스크린샷이 로딩 중 화면으로 찍히는 경우는?**
대부분 DOM 안정성 감지가 해결합니다. 특수한 페이지라면 "3초 더 기다린 후 스크린샷 찍어줘" 처럼 요청하면 AI가 `wait_ms=3000` 파라미터를 사용합니다.

**Q. 지도 스크린샷이 흰 화면으로 나오는 경우는?**
Canvas 자동 대기(4초)가 적용되지만 느린 네트워크에서는 부족할 수 있습니다. "5초 기다린 후 지도 스크린샷 찍어줘" 처럼 요청해보세요.

**Q. 안티봇 차단이 발생하면?**
일부 사이트는 자동화 접근을 차단합니다. 이 경우 `browser_get_text()`로 텍스트만 추출하여 내용을 요약하는 방식으로 우회할 수 있습니다.

**Q. Docker 환경에서 headed 모드를 사용하려면?**
Docker 컨테이너 내에서는 가상 디스플레이(Xvfb)가 필요합니다. 기본적으로 headless 모드를 사용하는 것을 권장합니다.
