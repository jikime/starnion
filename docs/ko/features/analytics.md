# 분석 & 사용량

## 개요

Starnion은 두 가지 분석 정보를 제공합니다.

1. **대화 분석 (Analytics)**: 채팅 패턴, 채널별 통계, 월별 추이
2. **LLM 사용량 (Usage)**: 토큰 소비량, 모델별 호출 횟수, 예상 비용

이 데이터를 통해 AI를 얼마나 활발히 사용하고 있는지, 어떤 채널을 주로 이용하는지, LLM API 비용이 얼마나 발생하는지 파악할 수 있습니다.

---

## 대화 분석 (Analytics)

**Settings > Analytics** 또는 API `GET /api/v1/analytics?user_id=<id>`로 조회합니다.

### 요약 통계

| 항목 | 설명 |
|------|------|
| 총 메시지 수 | 전체 기간 동안 주고받은 메시지 합계 |
| 이번 달 메시지 | 현재 월의 메시지 수 |
| 사용자 메시지 | 사용자가 보낸 메시지 수 |
| AI 메시지 | AI가 응답한 메시지 수 |
| 총 대화 수 | 생성된 대화(스레드) 수 |
| 일평균 메시지 | 최근 30일 기준 일평균 |
| 전월 대비 (MoM) | 이번 달 vs 지난 달 메시지 증감률 (%) |

### 채널별 통계

Telegram과 웹 채팅(WebChat)으로 구분된 사용량을 보여줍니다.

```
Telegram 메시지:  ████████████ 156건 (62%)
웹 채팅 메시지:   ███████      96건 (38%)
```

### 일별 메시지 추이

최근 30일간의 일별 메시지 수 추이를 그래프로 시각화합니다. 어느 요일에 가장 많이 사용하는지, 특정 이벤트 이후 사용량이 증가했는지 확인할 수 있습니다.

### 시간대별 분포

하루 중 어느 시간대에 가장 활발히 대화하는지 보여줍니다. 24시간 분포로 표시되며, 오전/오후/야간 사용 패턴을 파악할 수 있습니다.

### 주간 패턴

요일별 평균 메시지 수를 분석합니다. 업무일과 주말의 사용량 차이를 시각적으로 비교할 수 있습니다.

## LLM 사용량 (Usage)

**Settings > Usage** 또는 API `GET /api/v1/usage?user_id=<id>`로 조회합니다.

### 사용량 개요

모든 LLM 호출의 토큰 사용량을 기록합니다.

| 항목 | 설명 |
|------|------|
| 총 호출 수 | 전체 LLM API 호출 횟수 |
| 총 입력 토큰 | 프롬프트에 소비된 총 토큰 |
| 총 출력 토큰 | AI 응답으로 생성된 총 토큰 |
| 총 토큰 합계 | 입력 + 출력 |
| 예상 총 비용 | 토큰 단가 기반 USD 환산 금액 |

### 모델별 사용량 분석

각 LLM 모델의 사용 현황을 비교합니다.

```
모델               호출수    입력토큰    출력토큰    예상비용
──────────────────────────────────────────────────────────
gemini-2.0-flash   1,234    2,345,678    456,789    $0.23
gpt-4o-mini           89      123,456     45,678    $0.18
claude-haiku-3-5      12       34,567      8,901    $0.04
```

### 일별 사용량 추이

최근 30일간의 일별 토큰 사용량을 시각화합니다. 비용 급증이 발생한 날짜를 빠르게 파악할 수 있습니다.

### 스킬별 사용량

어떤 기능(스킬)에서 토큰을 많이 소비하는지 분석합니다.

```
스킬              호출수    총 토큰     비율
──────────────────────────────────────
일반 대화          892     1,234,567    54%
가계부 분석         234       456,789    20%
문서 요약            89       234,567    10%
웹 검색              56       123,456     5%
기타                 ...         ...      11%
```

## 비용 절감 팁

LLM API 비용을 줄이는 실용적인 방법입니다.

### 1. 작업에 맞는 모델 선택

모든 대화에 가장 강력한 모델을 쓸 필요는 없습니다.

| 작업 유형 | 권장 모델 | 이유 |
|----------|----------|------|
| 간단한 질문/답변 | Gemini 2.0 Flash | 빠르고 저렴 |
| 일상 대화 | GPT-4o-mini | 저렴하면서 충분히 유능 |
| 긴 문서 분석 | Gemini 1.5 Pro | 긴 컨텍스트 효율적 처리 |
| 복잡한 추론 | Claude Opus (필요할 때만) | 비용이 높으므로 꼭 필요한 경우만 |

### 2. Gemini 무료 티어 활용

Gemini API의 무료 할당량(일 1,500회)을 초과하지 않는 범위에서는 비용이 발생하지 않습니다. 가벼운 개인 사용이라면 무료로 운영 가능합니다.

### 3. 컨텍스트 길이 최적화

대화가 길어질수록 매 턴마다 전체 대화 이력이 토큰으로 소비됩니다. 새로운 주제를 시작할 때는 새 대화를 만드는 것이 비용 효율적입니다.

### 4. 문서 인덱싱 비용 주의

문서를 업로드하면 임베딩 생성에 토큰이 소비됩니다. 큰 파일을 자주 재인덱싱하면 비용이 늘어납니다.

## API 엔드포인트

### 대화 분석

```http
GET /api/v1/analytics?user_id=<uuid>
```

응답에는 요약 통계, 채널 분포, 일별 추이, 시간대별 분포, 요일별 패턴이 포함됩니다.

### LLM 사용량

```http
GET /api/v1/usage?user_id=<uuid>
```

응답에는 총 사용량 요약, 모델별 분류, 일별 추이, 스킬별 분류가 포함됩니다.
