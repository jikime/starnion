# 모델 설정

## 개요

모델 설정은 Starnion에서 사용할 AI 모델과 프로바이더를 구성하는 설정 페이지입니다. 다양한 LLM 프로바이더의 API 키를 등록하고, 대화/이미지 생성/임베딩 등 용도별로 최적의 모델을 지정할 수 있습니다.

**주요 특징:**
- 다중 LLM 프로바이더 지원: Google Gemini, OpenAI, Anthropic Claude, GLM/Z.AI, Ollama
- API 키 관리: 프로바이더별 API 키 등록 및 검증
- 용도별 모델 할당: 채팅, 이미지 생성, 임베딩 등 기능별 모델 지정
- 고급 파라미터 설정: temperature, max_tokens 등 세부 조정
- 커스텀 엔드포인트: OpenAI 호환 API 지원 (Ollama, vLLM 등)

## 프로바이더 설정

### API 키 등록

1. **설정 > 모델** 메뉴로 이동합니다.
2. 사용할 프로바이더를 선택합니다.
3. **API 키**를 입력합니다.
4. **연결 테스트** 버튼을 눌러 키가 유효한지 확인합니다.
5. **저장**을 클릭합니다.

> API 키는 암호화되어 저장됩니다. 한 번 저장된 키는 화면에 전체가 표시되지 않습니다.

### 커스텀 엔드포인트 설정

Ollama나 vLLM 등 OpenAI 호환 API를 사용하는 경우:

1. 프로바이더에서 **커스텀**을 선택합니다.
2. **엔드포인트 URL**을 입력합니다 (예: `http://localhost:11434/v1`).
3. API 키가 필요한 경우 함께 입력합니다.
4. 연결 테스트를 실행합니다.

## 모델 할당

기능별로 다른 모델을 할당하여 비용과 성능을 최적화할 수 있습니다.

### 할당 가능 기능

| 기능 | 설명 | 권장 모델 |
|------|------|----------|
| **채팅** | 일반 대화, 질문 응답 | Gemini 2.0 Flash, GPT-4o-mini |
| **리포트** | 주간/월간 분석 보고서 생성 | GPT-4o, Claude 3.5 Sonnet |
| **이미지 생성** | DALL-E, Gemini 이미지 생성 | DALL-E 3, Gemini Imagen |
| **임베딩** | 문서/대화 벡터 변환 | text-embedding-3-small |
| **요약** | 긴 문서 요약 | Gemini 1.5 Pro |

### 할당 방법

1. **설정 > 모델** 메뉴에서 **모델 할당** 탭을 클릭합니다.
2. 각 기능 옆의 드롭다운에서 모델을 선택합니다.
3. **저장**을 클릭하면 즉시 적용됩니다.

> 할당하지 않은 기능은 시스템 기본 모델을 사용합니다.

## 고급 파라미터

모델의 동작을 세부적으로 조정할 수 있습니다.

| 파라미터 | 설명 | 기본값 | 범위 |
|---------|------|-------|------|
| **temperature** | 응답의 창의성/랜덤성 | 0.7 | 0.0 ~ 2.0 |
| **max_tokens** | 응답 최대 토큰 수 | 4096 | 1 ~ 모델 한도 |
| **top_p** | 누적 확률 샘플링 | 1.0 | 0.0 ~ 1.0 |

### 파라미터 가이드

- **정확한 답변이 필요할 때**: temperature를 0.1~0.3으로 낮추세요.
- **창의적인 글쓰기**: temperature를 0.8~1.2로 높이세요.
- **긴 응답이 필요할 때**: max_tokens를 8192 이상으로 설정하세요.

## 팁 & FAQ

**Q. API 키를 여러 프로바이더에 등록해도 되나요?**
네. 여러 프로바이더의 API 키를 동시에 등록하고, 기능별로 다른 프로바이더의 모델을 할당할 수 있습니다. 예를 들어 채팅은 Gemini, 이미지는 OpenAI DALL-E를 사용할 수 있습니다.

**Q. Ollama로 완전 무료로 사용할 수 있나요?**
네. Ollama를 로컬에 설치하고 커스텀 엔드포인트로 연결하면 외부 API 비용 없이 사용 가능합니다. 다만 로컬 하드웨어 성능에 따라 응답 속도가 달라집니다.

**Q. 모델을 변경하면 기존 대화에 영향이 있나요?**
아닙니다. 모델 변경은 이후 새로운 대화부터 적용됩니다. 기존 대화 내역은 그대로 유지됩니다.

**Q. API 키가 유효하지 않다고 나와요.**
연결 테스트를 실행하여 키를 확인하세요. 프로바이더 사이트에서 키가 활성 상태인지, 잔액이 남아있는지 확인해 보세요.

**Q. 어떤 모델을 선택해야 할지 모르겠어요.**
처음이라면 Gemini 2.0 Flash를 추천합니다. 무료 티어가 있고 대부분의 일상 대화에 충분한 성능을 제공합니다.
