# 일기

## 개요

Starnion의 일기 기능은 단순한 텍스트 기록을 넘어, AI가 여러분의 감정과 일상을 이해하고 기억하는 개인 저널입니다. 대화하듯 오늘 있었던 일을 이야기하면 AI가 자동으로 감정을 분석해 기록하고, 나중에 "그때 기분이 어땠지?"라고 물어보면 다시 꺼내볼 수 있습니다.

> 일기는 AI 메모리의 핵심 원천입니다. 기록된 내용은 AI가 여러분을 더 잘 이해하는 데 활용됩니다.

---

## 일기 작성하기

### 웹에서 작성

1. 사이드바에서 **일기** 메뉴를 클릭합니다.
2. 오른쪽 상단의 **+ 새 일기** 버튼을 누릅니다.
3. 제목(선택), 내용, 감정 태그를 입력합니다.
4. **저장** 버튼을 클릭합니다.

날짜는 기본적으로 오늘 날짜로 설정되며, 달력 아이콘을 클릭해 과거 날짜로 변경할 수도 있습니다.

### 채팅으로 작성

채팅창에 자연스럽게 일상을 이야기하면 AI가 자동으로 일기로 기록합니다. 별도의 명령어 없이도 됩니다.

**예시 대화:**

> 사용자: "오늘 팀 발표가 생각보다 잘 됐어. 긴장했는데 다행이야."
>
> AI: "잘 됐다니 다행이에요! 긴장을 이겨내고 좋은 결과를 얻었네요. 오늘 일기에 기록해 뒀을게요 😊"

> 사용자: "오늘 일기 써줘 — 아침에 늦잠 자서 지각할 뻔했고, 점심에 새로 생긴 파스타집 갔는데 맛있었어."
>
> AI: "기록했어요! 파스타집이 맛있었군요 — 다음에 또 가고 싶겠네요. 오늘 하루 수고하셨어요."

**팁:** 금액이 포함된 내용("점심에 12,000원 썼어")은 가계부로 자동 분류되며 일기로는 기록되지 않습니다.

## 일기 구조

각 일기 항목은 다음 필드로 구성됩니다:

| 필드 | 설명 | 예시 |
|------|------|------|
| **날짜** | 일기가 쓰인 날짜 (YYYY-MM-DD) | 2025-03-08 |
| **제목** | 간략한 제목 (선택 사항) | 팀 발표 성공 |
| **내용** | 일기 본문 | 오늘 발표가 잘 됐다... |
| **감정** | AI가 분류하거나 직접 선택한 감정 | 기쁨 |
| **태그** | 주제별 분류 태그 | #직장 #성취 |

### 감정 분류

AI는 내용을 바탕으로 다음 5가지 감정 중 하나를 자동으로 분류합니다:

- **기쁨** — 성취감, 흥분, 설렘 ("발표 성공!", "합격했어!")
- **좋음** — 행복, 만족, 감사 ("여유로운 하루", "맛있는 저녁")
- **보통** — 특별한 감정 변화 없는 평범한 일상
- **피곤** — 지침, 졸림, 체력 소진 ("야근", "운동 후 녹초")
- **나쁨** — 슬픔, 화남, 실망, 스트레스 ("싸웠어", "실망스러워")

웹 에디터에서 직접 감정을 변경할 수도 있습니다.

## 일기 조회

### 목록 보기

웹의 일기 화면에서 전체 일기 목록을 최신순으로 확인할 수 있습니다. 한 페이지에 50개씩 표시되며, 페이지네이션으로 이전 일기를 탐색합니다.

### 날짜로 검색

월별 필터를 사용해 특정 달의 일기만 볼 수 있습니다:

- 상단 드롭다운에서 **연도**와 **월**을 선택
- 해당 월의 모든 일기가 날짜 역순으로 표시됩니다

### 채팅으로 조회

> 사용자: "지난주 일기 보여줘"
>
> AI: "지난주 일기예요:\n- 3/3 (월): 팀 미팅, 감정: 보통\n- 3/5 (수): 친구와 저녁, 감정: 좋음\n- 3/7 (금): 주간 마감, 감정: 피곤"

> 사용자: "3월 일기 전부 보여줘"
>
> AI: "3월 일기 목록이에요. 총 12개 항목이 있어요..."

## AI 분석

### 감정 패턴 파악

주간 리포트에는 해당 주의 감정 흐름이 포함됩니다. 일기 데이터를 기반으로 어떤 감정이 많았는지, 특정 요일에 패턴이 있는지 분석합니다.

**예시 분석 결과:**
```
이번 주 감정 요약:
- 기쁨 2회, 좋음 3회, 보통 1회, 피곤 1회
- 주중(월~금)에는 피곤함이 많았고, 주말에는 좋음이 많았어요.
- 지난주 대비 전반적인 감정 상태가 개선됐어요!
```

### 스트레스 많은 날 찾기

> 사용자: "스트레스 받았던 날 언제야?"
>
> AI: "감정이 '나쁨'으로 기록된 날들이에요:\n- 2025-02-14: 야근, 업무 압박\n- 2025-02-28: 중요 발표 실패\n- 2025-03-03: 팀원과 갈등"

## 기억으로 활용

일기에 기록된 내용은 AI의 장기 메모리에 저장됩니다. 이 덕분에 며칠, 몇 달이 지나도 관련 내용을 기억하고 맥락에 맞는 답변을 드릴 수 있습니다.

**메모리 활용 예시:**

> 사용자: (3개월 후) "내가 예전에 파스타집 갔다고 했던 거 기억해?"
>
> AI: "네! 지난 3월 8일에 새로 생긴 파스타집에 가셨는데 맛있다고 하셨어요."

**기억으로 저장되는 내용:**
- 일기에 언급된 장소, 사람, 사건
- 반복적으로 등장하는 감정 패턴
- 사용자가 중요하게 여기는 일들

**기억으로 저장되지 않는 내용:**
- 금액 정보 (가계부로 별도 분류)
- 단순 사실 나열 없이 감정이 없는 메모 (메모 기능 사용 권장)

## 예시 대화

```
사용자: 오늘 운동 처음으로 1시간 했어. 힘들었지만 뿌듯해.
AI: 대단해요! 처음으로 1시간 운동을 해냈군요. 그 뿌듯함이 느껴져요.
    오늘 일기에 '기쁨'으로 기록해 뒀어요. 내일도 화이팅! 💪

사용자: 지난달에 기분 좋았던 날 언제였어?
AI: 지난달(2월) 일기 중 '기쁨'이나 '좋음'으로 기록된 날들이에요:
    - 2/8: 친구와 오랜만에 만남
    - 2/15: 프로젝트 완료
    - 2/22: 가족 모임

사용자: 2월 일기 총 몇 개야?
AI: 2월에 총 18개의 일기가 기록돼 있어요.
    감정 분포: 기쁨 4회, 좋음 7회, 보통 4회, 피곤 2회, 나쁨 1회
```

## 팁 & FAQ

**Q. 일기와 메모는 어떻게 다른가요?**
일기는 감정과 일상을 기록하는 개인 저널로, AI가 감정을 분석하고 장기 기억에 저장합니다. 메모는 태그 기반의 간단한 정보 저장소로 감정 분석이 없습니다.

**Q. 하루에 일기를 여러 개 쓸 수 있나요?**
네, 같은 날짜로 여러 개의 일기를 작성할 수 있습니다. 채팅을 통해 하루 동안 여러 번 이야기해도 각각 별도의 일기로 기록됩니다.

**Q. 과거 날짜로 일기를 쓸 수 있나요?**
웹 에디터에서 날짜를 직접 변경해 과거 날짜로 일기를 작성할 수 있습니다. 채팅에서는 "어제 있었던 일인데..." 또는 날짜를 명시하면 해당 날짜로 기록됩니다.

**Q. 일기를 삭제하면 AI 기억에서도 사라지나요?**
일기 삭제 시 해당 항목은 목록에서 제거됩니다. AI 메모리에 이미 저장된 내용은 주간 리포트 생성 시점에 따라 일부 남아있을 수 있습니다.

**Q. 채팅에서 자동으로 일기로 기록되는 기준이 있나요?**
감정이나 일상 경험이 담긴 메시지는 자동으로 일기로 저장됩니다. 금액 정보는 가계부로, 기억해두어야 할 정보는 메모로 분류됩니다.
