## 개요

**Starnion**은 완전히 자체 호스팅(self-hosted)되는 개인용 AI 어시스턴트 플랫폼입니다. 모든 데이터와 AI 인터랙션이 여러분의 인프라에서 실행되며, 어떠한 데이터도 외부 서버로 전송되지 않습니다.

클라우드 AI 서비스의 편리함을 유지하면서도, 개인 정보와 데이터 주권을 완전히 제어할 수 있도록 설계되었습니다.

---

## 핵심 개념

### 개인용 AI 에이전트 (Personal AI Agent)

Starnion의 AI 에이전트는 단순한 챗봇이 아닙니다. LangGraph 기반의 그래프 워크플로우를 통해 복잡한 작업을 단계별로 처리하는 지능형 에이전트입니다.

- 다중 AI 제공자 지원 (Anthropic Claude, OpenAI GPT, Google Gemini, Z.AI)
- 스킬(Skill) 시스템을 통한 기능 확장
- 장기 메모리 및 컨텍스트 관리
- 도구 호출(Tool Call)을 통한 외부 서비스 연동

### 프라이버시 우선 (Privacy-first)

```
여러분의 데이터 = 여러분의 인프라에만 저장
```

- 모든 대화 기록이 자체 PostgreSQL 데이터베이스에 저장됩니다
- AI API 호출 시 필요한 최소한의 정보만 전송됩니다
- 파일, 이미지, 문서는 자체 MinIO 스토리지에 보관됩니다
- 제3자 분석 서비스나 추적 코드가 없습니다

### 자체 호스팅 (Self-hosted)

```
여러분의 서버에서 실행 = 완전한 제어권
```

- Docker Compose 한 명령으로 전체 스택을 실행할 수 있습니다
- 클라우드, 온프레미스, 홈 서버 어디서든 실행 가능합니다
- 인터넷 연결 없이도 로컬 AI 모델과 함께 사용할 수 있습니다 (향후 지원)
- 데이터 백업과 마이그레이션을 직접 제어합니다

## 주요 기능

| 기능 | 설명 |
|------|------|
| **멀티 AI 제공자** | Anthropic, OpenAI, Google Gemini, Z.AI 동시 지원 |
| **멀티 채널** | 웹 UI, Telegram, Discord를 통한 접근 |
| **스킬 시스템** | 날씨, 번역, 검색, 일정 등 30+ 내장 스킬 |
| **문서 처리** | PDF, DOCX 업로드 및 의미 검색 (pgvector) |
| **이미지 분석** | 이미지 업로드 및 AI 분석 |
| **오디오 처리** | 음성 메모 업로드 및 전사(transcription) |
| **웹 검색** | 실시간 웹 검색 통합 |
| **페르소나** | 사용자 정의 AI 성격 및 프롬프트 |
| **Docker 지원** | 단일 명령 배포 |
| **실시간 스트리밍** | WebSocket 기반 실시간 응답 스트리밍 |

```
┌─────────────────────────────────────────────────────────────┐
│                        Starnion                              │
│                                                             │
│  ┌──────────────┐    ┌───────────────┐    ┌─────────────┐  │
│  │      UI      │    │    Gateway    │    │    Agent    │  │
│  │  (Next.js)   │───▶│  (Go + REST)  │───▶│  (Python)   │  │
│  │   :3000      │    │    :8080      │    │   :50051    │  │
│  └──────────────┘    └──────┬────────┘    └──────┬──────┘  │
│                             │                    │         │
│                    ┌────────▼────────────────────▼──────┐  │
│                    │                                    │  │
│                    │  ┌──────────────┐ ┌────────────┐  │  │
│                    │  │  PostgreSQL  │ │   MinIO    │  │  │
│                    │  │ (+ pgvector) │ │ (Storage)  │  │  │
│                    │  │    :5432     │ │   :9000    │  │  │
│                    │  └──────────────┘ └────────────┘  │  │
│                    │         인프라 레이어               │  │
│                    └────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘

채널 통합:
  Telegram Bot ──▶ Gateway
  Discord Bot  ──▶ Gateway
  Web Browser  ──▶ UI ──▶ Gateway
```

### 컴포넌트 설명

| 컴포넌트 | 역할 | 기술 스택 |
|----------|------|-----------|
| **UI** | 웹 인터페이스 | Next.js 15, React 19, TypeScript |
| **Gateway** | REST API 서버 / WebSocket | Go 1.22+, Gin |
| **Agent** | AI 엔진 / gRPC 서버 | Python 3.13+, LangGraph, gRPC |
| **PostgreSQL** | 메인 데이터베이스 / 벡터 검색 | PostgreSQL 16 + pgvector |
| **MinIO** | 파일 스토리지 | MinIO (S3 호환) |

### 데이터 흐름

```
사용자 메시지
     │
     ▼
  UI (Next.js)
     │ HTTP / WebSocket
     ▼
  Gateway (Go)
     │ gRPC
     ▼
  Agent (Python / LangGraph)
     │
     ├──▶ AI API (Gemini / Claude / GPT)
     ├──▶ 스킬 실행 (날씨, 검색, 번역...)
     ├──▶ PostgreSQL (대화 저장 / 벡터 검색)
     └──▶ MinIO (파일 접근)
     │
     ▼
  스트리밍 응답 → Gateway → UI
```

## 누구를 위한 서비스인가?

Starnion은 다음과 같은 분들에게 적합합니다:

### 개인 사용자

- AI와의 대화를 클라우드 서비스에 맡기지 않고 직접 제어하고 싶은 분
- 개인 일기, 메모, 목표 관리를 AI와 함께 하고 싶은 분
- Telegram을 통해 어디서든 개인 AI 어시스턴트에 접근하고 싶은 분

### 개발자 / 기술 사용자

- 자체 AI 플랫폼을 구축하고 싶은 개발자
- 스킬 시스템을 통해 커스텀 기능을 추가하고 싶은 분
- AI 에이전트 아키텍처를 공부하고 싶은 분

### 소규모 팀

- 사내 AI 어시스턴트를 운영하고 싶은 팀
- 고객 데이터를 외부 서비스에 보내지 않으려는 비즈니스
- 여러 채널(웹, Telegram, Discord)을 통한 AI 접근이 필요한 팀

### 적합하지 않은 경우

- 서버 관리 없이 즉시 사용 가능한 클라우드 서비스를 원하는 분 → ChatGPT, Claude.ai 등을 추천합니다
- 수백 명 이상의 대규모 사용자를 처리해야 하는 경우 → 추가적인 스케일링 작업이 필요합니다
