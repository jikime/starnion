# 日程 & 提醒

## 概述

Starnion 的日程和提醒功能帮助您设置循环和一次性提醒，让您不再错过重要事项。只需在聊天中自然地告诉 AI——例如"每天早上8点提醒我锻炼"——提醒会立即注册。

除了您自己设置的个人提醒，还有系统生成的自动通知（预算超支警告、倒计时提醒、目标进度更新等）。所有通知都通过 Telegram 发送，也可以在网页聊天中查看。

> 每天最多发送3条通知，安静时段（KST 22:00-08:00）不发送通知。

> 通知未按预期到达？有关每条通知的技能要求、数据阈值和疲劳度规则等详细触发条件，请参阅[通知触发条件详解](./notification-conditions.md)页面。

## 创建提醒

### 通过聊天创建

最简便的方法。只需在聊天中告诉 AI 您想要的提醒。

```
用户：每天早上8点提醒我锻炼。
AI：  运动提醒已设置为每天8:00！
      "锻炼时间到了！💪"

用户：每周一上午9点提醒我制定每周计划。
AI：  提醒已设置为每周一09:00！
      "制定本周计划的时候到了！📅"

用户：每月25日下午6点提醒我交房租。
AI：  提醒已设置为每月25日18:00！
      "交房租的日子到了！🏠"

用户：3月20日上午10点提醒我看医生预约。
AI：  3月20日2025年上午10:00一次性提醒已设置！
      "今天上午10点有医生预约！🏥"
```

### 从日程管理菜单创建

进入顶部菜单 > **日程** 标签 > 点击 **添加提醒**，使用表单注册提醒。

## Cron 表达式

提醒日程使用 cron 表达式管理。您无需直接输入——只需在聊天中详细描述您的需求。

**常见模式示例：**

| 模式 | Cron表达式 | 描述 |
|------|------------|------|
| 每天上午8点 | `0 8 * * *` | 每天08:00 |
| 每天晚上9点 | `0 21 * * *` | 每天21:00 |
| 每周一上午9点 | `0 9 * * 1` | 周一09:00 |
| 每周五下午6点 | `0 18 * * 5` | 周五18:00 |
| 每月1号上午9点 | `0 9 1 * *` | 每月1日09:00 |
| 每月月底晚上8点 | `0 20 28-31 * *` | 每月28-31日20:00 |
| 工作日（周一-周五）上午7点 | `0 7 * * 1-5` | 工作日07:00 |
| 周末（周六、日）上午10点 | `0 10 * * 6,0` | 周六周日10:00 |
| 每小时整点 | `0 * * * *` | 每小时整点 |
| 每15分钟 | `*/15 * * * *` | 每15分钟 |

### 个人提醒（用户创建）

```
用户：每天晚上10点提醒我写日记。
AI：  日记提醒已设置为每天22:00！

用户：每隔一个周五下午2点提醒我提交定期报告。

```

### 系统自动通知

Starnion 自动发送的通知。满足条件时发送，无需额外设置。

| 通知 | 触发条件 | 时间 |
|------|----------|------|
| 每周报告 | 有支出记录的用户 | 每周一09:00 |
| 预算警告 | 分类预算超过90% | 每小时检查 |
| 每日支出摘要 | 当天有支出记录的用户 | 每天21:00 |
| 消费异常 | 支出相对平均突然激增 | 每3小时检查 |
| 不活跃提醒 | 3天以上未登录 | 每天20:00 |
| 月末摘要 | 每月28-31日 | 每天21:00 |
| 目标进度 | 有活跃目标的用户 | 每周三12:00 |
| 模式分析结果 | 有足够分析数据的用户 | 每天14:00 |
| 倒计时提醒 | 有注册倒计时的用户 | 每天08:00 |

### 系统通知开关控制

每个系统通知都可以单独启用或禁用。

**操作方法：**
1. 点击左侧菜单中的**日程（Schedules）**。
2. 选择**系统（System）**标签。
3. 点击要控制的通知右侧的**Switch开关**。
   - 开启（On）：满足条件时发送通知
   - 关闭（Off）：无论条件如何都不发送通知

| 系统通知 | 可切换 |
|----------|-------|
| 每周报告 | O |
| 预算警告 | O |
| 每日支出摘要 | O |
| 不活跃提醒 | O |
| 月末摘要 | O |
| 消费异常 | O |
| 模式分析结果 | O |
| 目标进度 | O |
| 倒计时提醒 | O |
| 后台分析任务 | X（始终运行） |

## Telegram 通知

所有通知都通过 Telegram 消息应用发送。您需要先连接 Telegram。

连接 Telegram 的方法：
1. 搜索 Starnion Telegram 机器人（`@starnion_bot`）
2. 发送 `/start` 命令
3. 记下连接码并在网页端关联您的账号

> 有关连接 Telegram 的详细说明，请参阅 [Telegram 频道](../channels/telegram.md) 文档。

## 工作原理

用户注册的日程以JSON格式存储在 `knowledge_base` 表中。系统每15分钟检查所有活跃日程，如果当前时间在计划时间的15分钟窗口内，则发送通知。

```
日程工作流程：

1. 注册 → AI解析聊天请求并将日程保存到knowledge_base
2. 检查 → 每15分钟查询所有活跃日程
3. 时间检查 → 验证当前时间是否在计划时间±15分钟窗口内
4. 疲劳检查 → 检查安静时段（22:00-08:00）和每日最大次数（3次）
5. 发送 → 通过Telegram发送消息
6. 更新 → 一次性 → 标记为"已完成"，循环 → 更新last_sent
```

### 日程类型

| 类型 | 描述 | 示例 |
|------|------|------|
| `one_time` | 在指定日期时间触发一次 | "3月20日上午10点提醒我看医生预约" |
| `recurring` | 每天、每周或每月重复 | "每周一上午9点提醒我制定每周计划" |

### 通知发送限制

为防止通知过多，以下情况会阻止通知：

- **安静时段**：KST 22:00（晚上10点）- 08:00（早上8点）
- **每日限制**：每天最多3条自动通知（不包括对用户请求的响应）
- **活跃聊天期间阻止**：如果用户在过去一小时内发送了聊天消息（视为活跃状态）

## 常见问题

**Q. 可以创建多少个提醒？**
A. 个人提醒目前没有数量限制。但是管理太多提醒会很繁琐，建议只注册实际需要的内容。

**Q. 不用Telegram也能收到通知吗？**
A. Telegram通知需要连接Telegram。AI可能会在网页聊天中提及提醒，但主动推送通知只能通过Telegram实现。

**Q. 可以精确到分钟设置提醒时间吗？**
A. 可以。您可以指定分钟级精度，例如"每天早上8:30提醒我"。

**Q. 可以关闭系统自动通知（预算警告、消费异常等）吗？**
A. 可以。进入**日程菜单 > 系统标签**，可以单独开启或关闭每个通知。发送通知的9个系统调度器（每周报告、预算警告、每日摘要等）各有一个Switch开关。后台分析任务（对话分析、模式分析等）没有开关，始终运行。

**Q. 可以为特定日期设置一次性提醒吗？**
A. 可以。指定特定日期，如"3月20日上午10点提醒我"，会注册一次性提醒。该时间过后会自动停用。
