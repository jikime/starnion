# 报告

## 概述

Starnion 的报告功能使用 AI 全面分析您的日记条目、账单记录、目标和消费模式，然后自动生成每周和每月摘要报告。

每周一韩国标准时间上午9:00，前一周的活动摘要会自动通过 Telegram 发送给您。您也可以随时在聊天中输入"生成每周报告"来按需生成报告。所有生成的报告——包括历史报告——都可以在报告菜单中查看。

> 报告是您每周和每月反思的起点。AI 会先整理您的数据，请查阅内容并用其规划接下来的一周。

---

## 报告类型

| 类型 | 关键字 | 描述 | 自动生成时间（KST） |
|------|--------|------|---------------------|
| **每周报告** | `weekly` | 过去7天的全面分析 | 每周一09:00 |
| **每日摘要** | `daily` | 当天支出和活动摘要 | 每天21:00（有记录的日期） |
| **每月报告** | `monthly` | 当月完整分析 | 每月最后一天21:00 |
| **消费异常** | `anomaly` | 支出超过平均值200%的提醒 | 每3小时自动检查 |
| **模式洞见** | `pattern` | 个性化消费和行为模式洞见 | 每天14:00（满足条件时） |
| **目标进度** | `goal` | 活跃目标的进度摘要 | 每周三12:00 |

## 自动发送的报告

### 每周报告 — 每周一KST 09:00

每周一早上9:00，分析过去一周活动的报告会通过 Telegram 自动发送。仅发送给过去30天内有账单记录的用户。

**每周报告示例：**
```
📊 2025年3月第2周 — 每周报告

💰 本周总支出：187,400韩元

按分类支出：
🍽 餐饮：         78,500韩元  (41.9%)
🚌 交通：         24,200韩元  (12.9%)
☕ 咖啡馆：       18,700韩元  (10.0%)
🛒 购物：         42,000韩元  (22.4%)
🎬 休闲：         24,000韩元  (12.8%)

较上周：+12,300韩元（▲7.0%）
月累计：312,800韩元

💡 洞见
咖啡馆支出比上周增加35%。
餐饮支出已达预算的87%（剩余90,000韩元）。
```

### 每日摘要 — 每天KST 21:00

发送给当天有记录支出的用户。

```
📋 每日摘要 — 2025年3月8日

今日支出：42,300韩元
- 早餐：  6,500韩元
- 午餐：  9,000韩元
- 咖啡馆：4,500韩元
- 交通：  4,200韩元
- 晚餐：  18,100韩元

本周累计：127,400韩元 / 300,000韩元 (42.5%)
```

### 每月报告 — 每月最后一天KST 21:00

在每月最后一天晚上发送，包含对该月支出的完整分析。发送给当月有记录的用户。

### 消费异常 — 每3小时自动检查

如果今天的支出超过30天日均值的200%，立即发送提醒。需要至少7天的累积数据才能激活。

**消费异常示例：**
```
⚠️ 检测到消费异常

今日总支出：185,000韩元
30天日均值：   31,200韩元  (593%)

您的支出明显高于平时。请查看一下！
```

### 模式洞见 — 每天KST 14:00

AI 学习您的消费模式，在满足条件时发送个性化洞见。模式分析每天早上6:00在后台运行。

```
💡 模式洞见

您周五晚上的支出是平常的2.3倍。
过去4周周五平均：48,200韩元
（工作日平均：21,000韩元）

考虑单独设置一个周五晚间预算如何？
```

### 目标进度 — 每周三KST 12:00

发送给有活跃目标的用户，包含每周进度更新。

```
🎯 目标进度（3月第3周）

✅ 阅读挑战：4/5本（80%）— 再读一本就完成了！
🔄 运动习惯：8/12次（67%）— 本周还需完成2次
💰 旅行储蓄：150,000 / 200,000韩元 (75%)

较上周整体完成率：74%（+8%）
本周继续加油！💪
```

## 手动生成报告

您可以立即生成报告，无需等待自动计划。

### 在聊天中请求

```
用户：生成每周报告。
AI：  我已生成本周报告。[显示报告内容]

用户：生成本月的月度报告。
AI：  我已生成3月月度报告。

用户：分析今天的消费模式。
AI：  我已生成模式洞见报告。

用户：告诉我储蓄目标进度。
AI：  我已生成目标进度报告。
```

### 从报告菜单生成

进入顶部菜单 > **报告** 标签 > 点击 **生成报告** 按钮，然后选择您想要的报告类型。生成大约需要20-60秒。

> 手动生成的报告也会保存到报告列表中，之后可以再次查看。

## 报告内容详情

### 每周报告（weekly）

- 本周总支出
- 按分类的支出金额和占比
- 与上周相比的增减
- 月累计总计
- 预算使用率（设置预算时）
- AI洞见和节省建议

### 每月报告（monthly）

- 本月总支出
- 按分类的月度明细
- 与上个月的比较
- 逐日支出趋势
- 消费最高和最低的日期
- 是否实现了储蓄目标

### 模式洞见（pattern）

```
💡 消费模式洞见（2025-03-08）

1. 咖啡馆消费模式
   工作日上午（9-11点）咖啡馆光顾频率高。
   每周平均咖啡馆支出：23,500韩元

2. 周末消费特征
   周末购物支出是工作日的3.2倍。
   建议周末前列好购物清单。

3. 节省机会
   将便利店购物改为超市购物
   每月可节省约15,000韩元。
```

### 目标进度（goal）

```
🎯 目标进度（2025-03-08）

[目标1] 储蓄100万韩元应急基金
当前：650,000 / 1,000,000韩元 (65%)
预计完成日期：2025年5月15日

[目标2] 将月餐饮支出控制在15万韩元以内
3月餐饮支出：89,200 / 150,000韩元 (59.5%)
本月实现目标的可能性：✅ 高
```

## 查看报告

### 报告列表

在顶部菜单 > **报告** 标签中，您可以按时间倒序查看所有以前生成的报告。一次显示最多20条；点击加载更多按钮可以查看额外的报告。

### 报告筛选

| 筛选 | 描述 |
|------|------|
| 全部 | 所有报告 |
| 每周 | 仅 `weekly` 类型 |
| 每月 | 仅 `monthly` 类型 |
| 模式 | 仅 `pattern` 类型 |
| 目标 | 仅 `goal` 类型 |
| 异常 | 仅 `anomaly` 类型 |

### 查看历史报告

```
用户：显示上个月的月度报告。
AI：  2025年2月月度报告：
      [显示2月报告内容]

用户：比较上周和本周的每周报告。
AI：  2月第4周 vs 3月第1周：
      总支出：175,200韩元 → 187,400韩元（+12,200韩元）
      最大增加：购物（+18,000韩元）
      最大减少：餐饮（-5,800韩元）
```

## 工作原理

```
报告生成流程：

1. 触发      → 调度程序（自动）或用户请求（手动）
2. 获取数据  → 检索日记、账单、目标和备忘录数据
3. AI分析    → 通过gRPC向智能体发送分析请求（最长120秒）
4. 内容生成  → LLM生成全面的报告文本
5. 存储      → 永久保存在报告表中
6. 发送      → 在网页端显示 + 通过Telegram发送（如果已连接Telegram）
```

各类型报告标题格式：

| 类型 | 标题示例 |
|------|----------|
| `daily` | `每日摘要 — 2025年03月08日` |
| `weekly` | `2025年3月第2周 — 每周报告` |
| `monthly` | `2025年3月月度报告` |
| `anomaly` | `2025-03-08 检测到消费异常` |
| `pattern` | `2025-03-08 模式洞见` |
| `goal` | `2025-03-08 目标进度` |

## API端点

### 列出报告

```
GET /api/v1/reports?user_id={user_id}&type={type}&limit=20&offset=0
```

| 参数 | 必填 | 描述 |
|------|------|------|
| `user_id` | 必填 | 用户ID |
| `type` | 可选 | 报告类型（daily, weekly, monthly, anomaly, pattern, goal） |
| `limit` | 可选 | 最大返回结果数（默认20） |
| `offset` | 可选 | 偏移量（默认0） |

**响应示例：**
```json
[
  {
    "id": 42,
    "report_type": "weekly",
    "title": "2025年3月第2周 — 每周报告",
    "created_at": "2025-03-10 09:00"
  }
]
```

### 获取报告详情

```
GET /api/v1/reports/{id}?user_id={user_id}
```

**响应示例：**
```json
{
  "id": 42,
  "report_type": "weekly",
  "title": "2025年3月第2周 — 每周报告",
  "content": "📊 本周支出摘要...",
  "created_at": "2025-03-10 09:00"
}
```

### 手动生成报告

```
POST /api/v1/reports/generate
Content-Type: application/json

{
  "user_id": "...",
  "report_type": "weekly"
}
```

允许的 `report_type` 值：`daily`, `weekly`, `monthly`, `anomaly`, `pattern`, `goal`

## 自动发送计划摘要

| 报告 | 发送时间（KST） | 目标条件 |
|------|----------------|----------|
| 每周报告 | 每周一09:00 | 过去30天内有账单记录的用户 |
| 每日摘要 | 每天21:00 | 当天有支出记录的用户 |
| 每月报告 | 每月最后一天21:00 | 本月有记录的用户 |
| 消费异常 | 每3小时自动检查 | 支出超过日均值200%时 |
| 模式洞见 | 每天14:00 | 满足模式触发条件时 |
| 目标进度 | 每周三12:00 | 有活跃目标的用户 |

> 为防止通知过多，每天最多发送3条通知。
> 安静时段（KST 22:00-08:00）不发送通知。

## 常见问题

**Q. 我想在周一以外的日子收到每周报告。**
A. 自动每周报告目前固定在每周一上午9:00发送。如果您希望在其他时间收到，只需直接在聊天中请求"生成每周报告"。

**Q. 不用Telegram也能收到报告吗？**
A. 自动发送只能通过Telegram。在网页端，您可以直接从报告菜单生成报告或在聊天中请求。

**Q. 没有支出记录也会生成报告吗？**
A. 过去30天内没有支出记录的用户会被排除在自动每周报告发送之外。如果您手动请求，可能会生成"未找到记录"的报告。

**Q. 生成的报告会保留多久？**
A. 所有生成的报告会永久保留。您可以随时从报告菜单查看历史报告。

**Q. 报告生成太慢了。**
A. AI会分析您所有记录的数据，通常需要20-60秒。记录数量较多时处理时间可能增加。

**Q. 消费异常提醒太频繁了。**
A. 消费异常提醒每天发送次数不超过每日通知限制（3次）。如果您的日常支出较高且阈值感觉太低，继续积累账单记录——一旦30天平均值更好地反映您实际的消费模式，误报将会减少。
