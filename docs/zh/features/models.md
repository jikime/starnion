# 模型设置

## 概述

模型设置是用于配置 Starnion 中 AI 模型和提供商的设置页面。您可以注册各种 LLM 提供商的 API 密钥，并为聊天、图像生成、嵌入等不同用途分配最佳模型。

**主要特点：**
- 多 LLM 提供商支持：Google Gemini、OpenAI、Anthropic Claude、GLM/Z.AI、Ollama
- API 密钥管理：注册和验证各提供商的 API 密钥
- 按功能分配模型：为聊天、图像生成、嵌入等功能分别指定模型
- 高级参数设置：精细调整 temperature、max_tokens 等参数
- 自定义端点：支持 OpenAI 兼容 API（Ollama、vLLM 等）

## 提供商设置

### 注册 API 密钥

1. 进入 **设置 > 模型** 菜单。
2. 选择要使用的提供商。
3. 输入 **API 密钥**。
4. 点击 **测试连接** 按钮验证密钥是否有效。
5. 点击 **保存**。

> API 密钥以加密方式存储。保存后，密钥不会在界面上完整显示。

### 自定义端点设置

使用 Ollama 或 vLLM 等 OpenAI 兼容 API 时：

1. 在提供商中选择 **自定义**。
2. 输入 **端点 URL**（例如：`http://localhost:11434/v1`）。
3. 如需 API 密钥，一并输入。
4. 运行连接测试。

## 模型分配

为不同功能分配不同模型，以优化成本和性能。

### 可分配的功能

| 功能 | 说明 | 推荐模型 |
|------|------|---------|
| **聊天** | 日常对话、问答 | Gemini 2.0 Flash, GPT-4o-mini |
| **报告** | 周报/月报分析报告生成 | GPT-4o, Claude 3.5 Sonnet |
| **图像生成** | DALL-E、Gemini 图像生成 | DALL-E 3, Gemini Imagen |
| **嵌入** | 文档/对话向量转换 | text-embedding-3-small |
| **摘要** | 长文档摘要 | Gemini 1.5 Pro |

### 分配方法

1. 在 **设置 > 模型** 中点击 **模型分配** 标签。
2. 在各功能旁的下拉菜单中选择模型。
3. 点击 **保存** 即可立即生效。

> 未分配的功能将使用系统默认模型。

## 高级参数

精细调整模型行为。

| 参数 | 说明 | 默认值 | 范围 |
|------|------|-------|------|
| **temperature** | 回复的创意性/随机性 | 0.7 | 0.0 ~ 2.0 |
| **max_tokens** | 回复最大 token 数 | 4096 | 1 ~ 模型上限 |
| **top_p** | 累积概率采样 | 1.0 | 0.0 ~ 1.0 |

### 参数指南

- **需要精确答案时**：将 temperature 降低到 0.1~0.3。
- **创意写作**：将 temperature 提高到 0.8~1.2。
- **需要长回复时**：将 max_tokens 设置为 8192 或更高。

## 提示与常见问题

**Q. 可以为多个提供商注册 API 密钥吗？**
可以。您可以同时注册多个提供商的 API 密钥，并为不同功能分配不同提供商的模型。例如，聊天使用 Gemini，图像使用 OpenAI DALL-E。

**Q. 使用 Ollama 可以完全免费吗？**
可以。在本地安装 Ollama 并作为自定义端点连接，即可在无外部 API 费用的情况下使用。但响应速度取决于本地硬件性能。

**Q. 更改模型会影响现有对话吗？**
不会。模型更改仅适用于之后的新对话。现有对话历史将保持不变。

**Q. API 密钥显示为无效。**
运行连接测试来验证密钥。在提供商网站上检查密钥是否处于活跃状态以及余额是否充足。

**Q. 不知道该选择哪个模型。**
初次使用推荐 Gemini 2.0 Flash。它有免费层，对大多数日常对话提供足够的性能。
